Ανατομία Bash Script

Ένα Bash script είναι ένα αρχείο κειμένου που περιέχει εντολές shell, τρέχει από πάνω προς τα κάτω. Αντί να πληκτρολογείς κάθε εντολή χειροκίνητα, τις "αποθηκεύεις" — αυτή είναι η βάση του automation.

#!/bin/bash # ↑ shebang: λέει στο OS ποιον interpreter να χρησιμοποιήσει # script.sh — παράδειγμα NAME="Alice" # χωρίς spaces γύρω από το = echo "Hello, $NAME" if [ $? -eq 0 ]; then echo "OK" fi exit 0 # 0 = επιτυχία
⚠️ Πριν εκτελέσεις
Δώσε execute permission: chmod +x script.sh
Εκτέλεση: ./script.sh ή bash script.sh
🧠
Tip — shebang
Το #! λέγεται "shebang" (sharp + bang). Λέει στον kernel "τρέξε αυτό το script με /bin/bash". Χωρίς shebang, ο kernel δεν ξέρει ποιον interpreter να χρησιμοποιήσει.
source vs execute
# νέο subshell — αλλαγές δεν επηρεάζουν parent $ ./script.sh # τρέχει στο ΤΡΕΧΟΝ shell — αλλαγές ισχύουν $ source script.sh $ . script.sh # ίδιο με source
🧠
Tip
Χρήση source: .bashrc και config files. Αν βάλεις export VAR=x σε script και το τρέξεις με ./, δεν θα δεις το VAR στο shell σου. Με source θα το δεις.
📦

Μεταβλητές

Οι μεταβλητές αποθηκεύουν τιμές για να τις χρησιμοποιήσουμε αργότερα. Στο Bash δεν δηλώνεις τύπο — όλα αντιμετωπίζονται ως κείμενο. Δύο κρίσιμοι κανόνες: κανένα κενό γύρω από το = κατά τον ορισμό, και $ μπροστά κατά τη χρήση.

# ορισμός — χωρίς $ μπροστά, χωρίς spaces MY_VAR="hello world" COUNT=42 # χρήση — πάντα με $ echo "$MY_VAR" # ΠΑΝΤΑμε quotes! echo "${MY_VAR}s" # {} για concatenation # command substitution TODAY=$(date +%Y-%m-%d) # εκτελεί εντολή FILES=$(ls | wc -l) unset MY_VAR # διαγράφει τη μεταβλητή readonly MAX=100 # δεν μπορεί να αλλάξει
Quoting — κρίσιμο!
NAME="John Doe" # Double quotes: επεκτείνει μεταβλητές echo "Hello $NAME" # Hello John Doe # Single quotes: ΤΙΠΟΤΑ δεν επεκτείνεται echo 'Hello $NAME' # Hello $NAME (literal) # Χωρίς quotes: word splitting! echo $NAME # επικίνδυνο αν έχει spaces echo "$NAME" # πάντα έτσι ✓
🧠
Tip — κανόνας quoting
Double quotes = "ζωντανές" — μεταβλητές επεκτείνονται.
Single quotes = "παγωμένες" — τίποτα δεν αλλάζει, όλα literal.
Χρήσιμο μνημονικό: double = dynamic.
⭐

Special Variables — Οι πιο σημαντικές

Το Bash ορίζει αυτόματα ορισμένες "μαγικές" μεταβλητές που ενημερώνονται από το σύστημα κάθε φορά που εκτελείται μια εντολή. Μας επιτρέπουν να ξέρουμε: πόσα ορίσματα δόθηκαν, πώς πήγε η τελευταία εντολή, ποιο είναι το PID του script, κ.ά.

Variable Τι είναι Παράδειγμα χρήσης Tip
$$ PID του τρέχοντος script Temp files: /tmp/lock.$$ Δύο $ = "ο PID του εαυτού μου"
$? Exit status τελευταίας εντολής (0=OK) Αμέσως μετά command για έλεγχο ? = "πώς πήγε;"
$! PID τελευταίου background process prog & echo $! ! = "το τελευταίο !"
$0 Όνομα του script echo "Usage: $0 <arg>" 0 = ο "zeroth" argument
$1, $2… Positional arguments (1ο, 2ο…) ./script.sh foo bar → $1=foo Σειρά στη γραμμή εντολών
$# Πλήθος arguments που δόθηκαν if [ $# -ne 2 ] # = αριθμός (hash = count)
$@ Όλα τα arguments (ξεχωριστά) for arg in "$@" @ = "all at" — όλα μαζί αλλά ξεχωριστά
$* Όλα τα arguments (ένα string) Σπάνια χρήση * = "glob" = ένα μπλοκ
🧠
Tip — $@ vs $* μέσα σε double quotes
"$@" → "arg1" "arg2" "arg3" (κάθε arg ξεχωριστά — ΑΥΤΟ χρησιμοποίησε)
"$*" → "arg1 arg2 arg3" (όλα ένα string — σπάει αν args έχουν spaces)
Παράδειγμα: αν arg1 = "hello world", με "$@" παραμένει ένα arg, με "$*" σπάει σε δύο.
🎯

Arguments & Input

Τα arguments είναι τιμές που δίνουμε στο script κατά την εκτέλεση: ./script.sh arg1 arg2. Εναλλακτικά, μπορούμε να ζητάμε δεδομένα διαδραστικά κατά τη διάρκεια της εκτέλεσης με την εντολή read.

#!/bin/bash # ελεγχω αν δοθηκαν σωστα args if [ $# -ne 2 ]; then echo "Usage: $0 <src> <dst>" >&2 exit 1 fi SRC="$1" DST="$2" # επαναλαμβανω ολα τα arguments for arg in "$@"; do echo "Arg: $arg" done
# διαβασμα από χρήστη read -p "Δώσε όνομα: " NAME echo "Hello, $NAME" # read με timeout read -t 5 -p "Input (5s): " X # read αρχείο γραμμή-γραμμή while read -r line; do echo "Line: $line" done < "input.txt"
🧠
Tip — read -r
Το -r = raw. Χωρίς -r, τα backslashes (\) ερμηνεύονται ως escape sequences. Σχεδόν πάντα χρησιμοποίησε read -r.

Συνθήκες — if / case

Οι συνθήκες επιτρέπουν στο script να παίρνει αποφάσεις. Η σύνταξη στο Bash είναι αυστηρή: κάθε κενό γύρω από τις αγκύλες [ ] είναι υποχρεωτικό, γιατί το [ είναι στην πραγματικότητα μια εντολή!

# if-elif-else if [ "$1" == "hello" ]; then echo "greeting" elif [ "$1" == "bye" ]; then echo "farewell" else echo "unknown" fi # έλεγχος αρχείου if [ -f "$FILE" ]; then echo "file exists" fi # λογικοί τελεστές if [ -f "$F" ] && [ -r "$F" ]; then echo "exists and readable" fi
Operators [ ] — Πλήρης Λίστα
String Numeric File
== != -eq -ne -f (file)
-z (κενό) -lt (less) -d (dir)
-n (μη κενό) -gt (greater) -e (exists)
< > (lex) -le -ge -r -w -x
🧠
Tip — αριθμητικοί operators
Χρησιμοποιούν "λέξεις" αντί συμβόλων: -eq = equal, -ne = not equal, -lt = less than, -gt = greater than, -le/-ge = less/greater or equal.
# case statement case "$1" in "start") echo "starting..." ;; "stop"|"quit") # OR με | echo "stopping..." ;; *.txt) # pattern matching echo "text file" ;; *) # default echo "unknown" ;; esac
⚠️ Κλασικά λάθη με [ ]
Πάντα spaces γύρω από [ και ]: [ $x -eq 5 ] ✓ — [$x -eq 5] ✗
Πάντα quotes γύρω από string variables: [ "$NAME" == "Alice" ] — χωρίς quotes σπάει αν έχει spaces.
🔄

Βρόχοι — for / while / until

# for με list for i in 1 2 3 4 5; do echo "Number: $i" done # for με range {start..end} for i in {1..10}; do echo $i done # for C-style for (( i=0; i<10; i++ )); do echo $i done # for σε αρχεία (glob) for f in *.c; do echo "Compiling: $f" done
# while loop COUNT=0 while [ $COUNT -lt 5 ]; do echo "Count: $COUNT" COUNT=$(( COUNT + 1 )) done # infinite loop με break while true; do read -p "Input (q=quit): " X [ "$X" == "q" ] && break echo "Said: $X" done # until = αντίστροφο while (τρέχει ΕΩΣ ότου) until [ $COUNT -ge 10 ]; do COUNT=$(( COUNT + 1 )) done
🧠
Tip — while vs until
while τρέχει ΕΝΩ η συνθήκη είναι αληθής. until τρέχει ΜΕΧΡΙ η συνθήκη να γίνει αληθής. Ίδια λογική, αντίστροφη συνθήκη.
break = σταμάτα τον βρόχο · continue = παράλειψε αυτή την επανάληψη.
break & continue
break = βγες από τον βρόχο · continue = πήγαινε στην επόμενη επανάληψη · break 2 = βγες από 2 επίπεδα nested loops

Αριθμητικές Πράξεις

Παρόλο που το Bash είναι κυρίως εργαλείο κειμένου, υποστηρίζει βασικά μαθηματικά. Προσοχή: δουλεύει μόνο με ακέραιους. Για δεκαδικούς χρειάζεσαι εξωτερικό εργαλείο όπως bc ή awk.

# 1. $(( )) — η σύγχρονη προσέγγιση A=10; B=3 SUM=$(( A + B )) # 13 DIV=$(( A / B )) # 3 (integer! όχι 3.33) MOD=$(( A % B )) # 1 (υπόλοιπο) POW=$(( A ** 2 )) # 100 (δύναμη) (( A++ )) # increment (( A += 5 )) # A = A + 5 # σύγκριση με (( )) if (( A > B )); then echo "A > B" fi
# 2. let — παλιότερο let "RESULT = A * 2 + 1" # 3. expr — για παλιά συστήματα SUM=$(expr $A + $B) MUL=$(expr $A \* $B) # escape το * # 4. bc — για δεκαδικούς! PI=$(echo "scale=4; 22/7" | bc) echo $PI # 3.1428 SQRT=$(echo "sqrt(2)" | bc -l) echo $SQRT # 1.41421356...
🧠
Tip — integer-only!
Bash κάνει μόνο integer arithmetic. $(( 7/2 )) = 3, ΟΧΙ 3.5. Για δεκαδικούς πάντα bc. Το scale=N στο bc ορίζει δεκαδικά ψηφία.
⚠️ Σε θυμίζω
Με expr, το * πρέπει να escape-αριστεί: expr $A \* $B. Αλλιώς ο shell το ερμηνεύει ως glob expansion.
⚙️

Functions

#!/bin/bash # ορισμός (δύο συντακτικοί τρόποι, ίδιο αποτέλεσμα) function greet() { echo "Hello, $1!" # $1 = πρώτο arg της function } say_bye() { echo "Bye, $1" return 0 # exit status (0-255) } # κλήση — πάντα ΜΕΤΑ τον ορισμό greet "Alice" say_bye "Bob"
# local variables X="global" my_func() { local X="local" # δεν επηρεάζει global X echo "inside: $X" # local } my_func echo "outside: $X" # global # επιστροφή τιμής: echo + $() add() { echo $(( $1 + $2 )) # "επιστρέφω" με echo } RESULT=$(add 3 5) # "πιάνω" με $() echo $RESULT # 8
🧠
Tip — return vs echo
Το return N σε bash επιστρέφει exit status (0-255), ΟΧΙ δεδομένα όπως στη C. Για να επιστρέψεις δεδομένα (string, αριθμό), χρησιμοποίησε echo και πίασε με $().
🚨 Πρέπει να ορίζεται πριν καλεστεί!
Σε αντίθεση με C, οι functions σε bash πρέπει να οριστούν πριν κληθούν. Αν καλέσεις function που δεν έχει οριστεί ακόμα, bash error.
✂️

String Manipulation

Parameter Expansion — οι πιο χρήσιμες
F="hello_world.txt" # μήκος string echo ${#F} # 15 # αφαίρεση suffix (% = από δεξιά) echo ${F%.txt} # hello_world ← πολύ χρήσιμο! echo ${F%.*} # hello_world (αφαιρεί extension) # αφαίρεση prefix (# = από αριστερά) echo ${F#hello_} # world.txt echo ${F##*/} # μόνο filename (αφαιρεί path) # replace echo ${F/world/bash} # hello_bash.txt # substring ${var:start:length} echo ${F:0:5} # hello echo ${F:6} # world.txt (από pos 6)
🧠
Tip — % και # για αφαίρεση
% αφαιρεί από τα δεξιά (suffix). # αφαιρεί από τα αριστερά (prefix). Διπλά (%% και ##) = greedy (αφαιρεί όσο γίνεται).
Μνημονικό: % μοιάζει σαν να "κόβει" την ουρά.
