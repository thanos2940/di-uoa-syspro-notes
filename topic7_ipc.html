1. System V IPC Overview

Τα System V IPC mechanisms επεκτείνουν τις δυνατότητες επικοινωνίας μεταξύ διεργασιών πέρα από τα pipes. Ενώ τα pipes είναι απλά, έχουν σημαντικούς περιορισμούς. Το System V IPC παρέχει 3 μηχανισμούς: Message Queues, Shared Memory, Semaphores.

Pipe Limitations
  • Unidirectional: Μονόδρομα — χρειάζεσαι 2 pipes για αμφίδρομη επικοινωνία
  • Related processes: Anonymous pipes λειτουργούν μόνο μεταξύ parent-child
  • No message boundaries: Byte stream — δεν ξέρεις πού τελειώνει ένα μήνυμα
  • Process lifetime: Καταστρέφονται μαζί με τη διεργασία
  • No priorities: FIFO μόνο — δεν μπορείς να δώσεις προτεραιότητα
Three IPC Mechanisms
MechanismPurpose
Message QueuesΑνταλλαγή δομημένων μηνυμάτων με τύπο/προτεραιότητα
Shared MemoryΚοινόχρηστη μνήμη — ο ταχύτερος IPC μηχανισμός
SemaphoresSynchronization — έλεγχος πρόσβασης σε κοινούς πόρους
System V IPC Characteristics
  • Kernel Lifetime: Τα IPC objects ζουν στον kernel — δεν καταστρέφονται όταν πεθάνουν οι διεργασίες! Πρέπει να τα σβήσεις ρητά (ipcrm) ή με IPC_RMID.
  • Key-based: Κάθε IPC object αναγνωρίζεται από ένα key_t — ένα integer key. Οι unrelated processes μπορούν να βρουν τον ίδιο πόρο χρησιμοποιώντας το ίδιο key.
  • No file descriptors: Δεν χρησιμοποιούν fd — δεν μπαίνουν στον file descriptor table, δεν έχουν inodes.
  • Access permissions: Κάθε IPC object έχει permissions (owner, group, mode) — όπως τα αρχεία.
🧠
Tip — ipcs & ipcrm commands
ipcs: Δείχνει τα ενεργά IPC resources (message queues, shared memory, semaphores).
ipcrm -q <id>: Σβήνει message queue. ipcrm -m <id>: Σβήνει shared memory. ipcrm -s <id>: Σβήνει semaphore set.
Exam tip: Αν ξεχάσεις να σβήσεις IPC objects, μένουν στον kernel μέχρι reboot ή ipcrm!

2. IPC Keys & Access Control

Κάθε IPC object χρειάζεται ένα key (key_t) για δημιουργία/σύνδεση. Υπάρχουν 3 τρόποι: ftok() (recommended), IPC_PRIVATE (parent-child), ή hard-coded integer (bad practice).

ftok() — Generate Key
key_t ftok(const char *pathname, int proj_id); // pathname: πρέπει να υπάρχει (χρησιμοποιεί inode) // proj_id: 8-bit integer (non-zero) // Returns: key_t or -1 on error // Example: key_t key = ftok("/tmp/myfile", 'A'); if (key == -1) { perror("ftok"); exit(1); }
How ftok works
Συνδυάζει τον inode number του αρχείου + proj_id byte + device number → μοναδικό key. Αν δύο unrelated processes καλέσουν ftok με τα ίδια arguments, παίρνουν το ίδιο key.
IPC_PRIVATE
// Creates a unique key (guaranteed new object) int id = msgget(IPC_PRIVATE, 0666); // Only parent + children can access (must pass ID)

Χρήσιμο όταν μόνο parent-child χρειάζονται πρόσβαση. Ο parent δημιουργεί, και τα children κληρονομούν τον ID μέσω fork().

Access Permissions — struct ipc_perm
struct ipc_perm {
  uid_t uid; // owner's effective user ID
  gid_t gid; // owner's effective group ID
  uid_t cuid; // creator's effective user ID
  gid_t cgid; // creator's effective group ID
  mode_t mode; // permissions (read/write)
};

Τα permissions δηλώνονται σαν file permissions: 0666 = read/write for all. Η IPC_CREAT flag δημιουργεί αν δεν υπάρχει. Η IPC_EXCL + IPC_CREAT = error αν ήδη υπάρχει.

Common Flags Pattern
// Create new (or get existing) id = msgget(key, IPC_CREAT | 0666); // Create new (FAIL if exists) id = msgget(key, IPC_CREAT | IPC_EXCL | 0666); // Get existing (FAIL if doesn't exist) id = msgget(key, 0);

3. Message Queues API

Τα Message Queues επιτρέπουν ανταλλαγή δομημένων μηνυμάτων μεταξύ διεργασιών. Κάθε μήνυμα έχει ένα type (long integer), που επιτρέπει selective receiving — κάτι αδύνατο με pipes!

msgget() — Create/Get Queue
#include <sys/msg.h> int msgget(key_t key, int msgflg); // Returns: queue ID (>= 0) or -1 on error
Message Structure
struct message {
  long mtype; // message type (MUST be > 0)
  char mtext[1]; // message data (flexible)
};
Flexible Message Body
Το mtext μπορεί να είναι οποιοδήποτε μέγεθος. Ορίζεις δικό σου struct με μεγαλύτερο array ή επιπλέον πεδία. Το mtype πρέπει πάντα να είναι θετικό.
msgsnd() — Send Message
int msgsnd(int msqid, const void *msgp, // pointer to message struct size_t msgsz, // size of mtext (NOT entire struct!) int msgflg); // IPC_NOWAIT or 0 // Returns: 0 on success, -1 on error
msgsz = sizeof data ONLY
Πέρνα μέγεθος μόνο του mtext μέρους, ΟΧΙ ολόκληρου του struct! Δηλαδή sizeof(msg) - sizeof(long).
msgrcv() — Receive Message
ssize_t msgrcv(int msqid, void *msgp, // output buffer size_t msgsz, // max size of mtext long msgtyp, // which message type int msgflg); // IPC_NOWAIT or 0 // Returns: number of bytes received, or -1
msgtyp Values — Selective Receive
msgtypBehavior
= 0Πρώτο μήνυμα στην ουρά (FIFO)
> 0Πρώτο μήνυμα με αυτό ακριβώς το type
< 0Πρώτο μήνυμα με type ≤ |msgtyp| (μικρότερο ή ίσο)
msgctl() — Control Operations
int msgctl(int msqid, int cmd, struct msqid_ds *buf); // cmd = IPC_STAT : read queue info into buf // cmd = IPC_SET : modify permissions // cmd = IPC_RMID : delete queue (buf=NULL)
🧠
Exam Tip — msgtyp < 0
Με msgtyp = -3, λαμβάνεις το πρώτο μήνυμα με type 1, 2, ή 3 (ό,τι υπάρχει πρώτο). Αυτό φτιάχνει priority queue: μικρότερα types = υψηλότερη προτεραιότητα!

4. Message Queue Examples

Παραδείγματα server-client αρχιτεκτονικής και priority queue χρησιμοποιώντας message queues. Δείχνουν πώς unrelated processes ανταλλάσσουν μηνύματα.

Server — Creates Queue & Receives
#define MSGKEY 1234 struct message { long mtype; char mtext[64]; }; int main() { int msqid = msgget(MSGKEY, 0666 | IPC_CREAT); struct message msg; while (1) { msgrcv(msqid, &msg, sizeof(msg.mtext), 0, 0); // type=0: any message printf("Received [type=%ld]: %s\n", msg.mtype, msg.mtext); } msgctl(msqid, IPC_RMID, NULL); return 0; }
Client 1 — Sends type=1 messages
int main() { int msqid = msgget(MSGKEY, 0); // get existing struct message msg; msg.mtype = 1; strcpy(msg.mtext, "Hello from client 1"); msgsnd(msqid, &msg, sizeof(msg.mtext), 0); return 0; }
Priority Queue — Using msgtyp

Χρησιμοποιώντας negative msgtyp, μπορούμε να φτιάξουμε priority queue:

// q.h — Shared header #define MAXOBN 50 #define MAXPRIOR 10 struct message { long mtype; // 1=highest, MAXPRIOR=lowest char mtext[MAXOBN]; }; // enter.c — Add to queue with priority msg.mtype = priority; // lower = higher priority msgsnd(qid, &msg, sizeof(msg.mtext), 0); // serve.c — Get highest priority first msgrcv(qid, &msg, sizeof(msg.mtext), -MAXPRIOR, 0); // ← negative! // Gets lowest-numbered type first = highest priority
Message Queues vs Pipes
FeatureMsg QueuePipe
Message boundariesYes (typed messages)No (byte stream)
PriorityYes (via msgtyp)No (FIFO only)
Unrelated processesYes (via key)Only named pipes
LifetimeKernel (persists!)Process
DirectionBidirectionalUnidirectional

5. Shared Memory API

Η Shared Memory είναι ο ταχύτερος IPC μηχανισμός γιατί τα δεδομένα δεν αντιγράφονται μεταξύ user space και kernel space. Οι διεργασίες βλέπουν απευθείας τις ίδιες physical pages μνήμης. Αλλά χρειάζεται synchronization (semaphores/mutexes).

shmget() — Create/Get Segment
#include <sys/shm.h> int shmget(key_t key, size_t size, // bytes int shmflg); // IPC_CREAT | perms // Returns: shared memory ID or -1
shmat() — Attach Segment
void *shmat(int shmid, const void *shmaddr, // NULL = kernel picks int shmflg); // 0 or SHM_RDONLY // Returns: pointer to segment or (void *)-1
How it works
Η shmat() "κολλάει" το shared memory segment στο address space της διεργασίας. Μετά, η διεργασία μπορεί να γράψει/διαβάσει σαν κανονική μνήμη (ποίντερ).
shmdt() — Detach Segment
int shmdt(const void *shmaddr); // "Αποκολλά" τη shared memory // ΔΕΝ σβήνει τα δεδομένα!
shmctl() — Control Operations
int shmctl(int shmid, int cmd, struct shmid_ds *buf); // IPC_STAT : read segment info // IPC_SET : modify permissions // IPC_RMID : mark for deletion // (actual deletion when shm_nattch reaches 0)
struct shmid_ds
struct shmid_ds {
  struct ipc_perm shm_perm; // permissions
  size_t shm_segsz; // segment size (bytes)
  pid_t shm_cpid; // PID of creator
  pid_t shm_lpid; // PID of last shmat/shmdt
  shmatt_t shm_nattch; // # current attaches
  time_t shm_atime; // last attach time
  time_t shm_dtime; // last detach time
};
── Shared Memory Lifecycle ──
shmget() → create/get
↓
shmat() → attach to address space
↓
read/write via pointer
↓
shmdt() → detach
↓
shmctl(IPC_RMID) → delete

6. Shared Memory Examples

sharedMem1.c — Create & Attach (IPC_PRIVATE)
int main() { int shmid; char *ptr; // Create shared memory (64 bytes) shmid = shmget(IPC_PRIVATE, 64, IPC_CREAT | 0666); if (shmid == -1) { perror("shmget"); exit(1); } printf("Shared Memory ID: %d\n", shmid); // Attach to our address space ptr = (char *) shmat(shmid, NULL, 0); if (ptr == (char *) -1) { perror("shmat"); exit(1); } // Write data strcpy(ptr, "Hello shared memory!"); printf("Data: %s\n", ptr); // Detach (don't delete — other process can use it) shmdt(ptr); return 0; }
sharedMem2.c — Attach by ID & Read
int main(int argc, char *argv[]) { int shmid = atoi(argv[1]); // ID from cmdline char *ptr; // Attach to existing segment ptr = (char *) shmat(shmid, NULL, 0); if (ptr == (char *) -1) { perror("shmat"); exit(1); } // Read what was written printf("Read: %s\n", ptr); // Modify strcat(ptr, " - modified!"); // Detach and delete shmdt(ptr); shmctl(shmid, IPC_RMID, NULL); return 0; }
No synchronization!
Αυτά τα παραδείγματα δεν χρησιμοποιούν synchronization. Αν δύο διεργασίες γράφουν ταυτόχρονα → race condition! Χρειάζονται semaphores (δες §8-9).
🧠
Tip — SHM vs Message Queues
Shared Memory: Ταχύτατο (zero-copy), αλλά χρειάζεται manual synchronization.
Message Queues: Πιο αργό (kernel copies), αλλά built-in synchronization (msgsnd/msgrcv block).
Χρησιμοποίησε shared memory + semaphores για performance-critical εφαρμογές.

7. Semaphore Concepts

Ένας semaphore είναι ένας μη-αρνητικός counter που χρησιμοποιείται για synchronization. Δύο ατομικές πράξεις: UP (V) = αύξηση, DOWN (P) = μείωση (μπλοκάρει αν ο counter είναι 0).

UP & DOWN Operations
UP (V / signal / release):
  semaphore_value++
  Αν κάποιος περιμένει → ξύπνησέ τον

DOWN (P / wait / acquire):
  if (semaphore_value > 0)
    semaphore_value--
  else
    BLOCK until value becomes > 0
Naming Convention
P = Proberen (Dutch: "to test") = DOWN = wait.
V = Verhogen (Dutch: "to increment") = UP = signal.
Dijkstra's original naming. Εναλλακτικά: wait/signal, acquire/release.
Semaphore Properties
  • Advisory: Δεν εμποδίζουν φυσικά την πρόσβαση. Οι διεργασίες πρέπει να συνεργάζονται (σαν τα traffic lights — δεν σταματούν αυτοκίνητα φυσικά).
  • Atomic: Οι UP/DOWN πράξεις εκτελούνται ατομικά από τον kernel.
  • Binary semaphore (0/1): Ισοδύναμο με mutex — mutual exclusion.
  • Counting semaphore: Τιμή > 1 — ελέγχει πρόσβαση σε N πόρους (π.χ. N tape drives).
  • Blocking: Αν DOWN σε semaphore=0, η διεργασία κοιμάται μέχρι κάποιος κάνει UP.
Mutual Exclusion with Semaphore (init=1)
// semaphore S initialized to 1 DOWN(S); // S: 1→0 (enter critical section) /* critical section */ UP(S); // S: 0→1 (leave critical section)

Αν ένα δεύτερο process κάνει DOWN ενώ S=0, θα μπλοκάρει μέχρι το πρώτο κάνει UP.

8. System V Semaphore API

Τα System V semaphores λειτουργούν σε sets (σύνολα). Ένα semget() δημιουργεί ένα set από πολλαπλά semaphores. Η semop() μπορεί να κάνει ατομικές πράξεις σε πολλαπλά semaphores ταυτόχρονα.

semget() — Create/Get Semaphore Set
#include <sys/sem.h> int semget(key_t key, int nsems, // # semaphores in set int semflg); // IPC_CREAT | perms // Returns: semaphore set ID or -1

Π.χ. semget(key, 3, IPC_CREAT|0666) δημιουργεί ένα set με 3 semaphores (αριθμημένοι 0, 1, 2).

semop() — Semaphore Operations
int semop(int semid, struct sembuf *sops, // array of operations size_t nsops); // # operations
struct sembuf {
  unsigned short sem_num; // which semaphore (index)
  short sem_op; // operation value
  short sem_flg; // IPC_NOWAIT, SEM_UNDO
};
sem_op Values
sem_opAction
> 0UP: Add sem_op to semval (release resources)
< 0DOWN: Subtract |sem_op| from semval. If semval < |sem_op|, BLOCK until possible.
= 0WAIT-FOR-ZERO: Block until semval becomes 0.
semctl() — Control Operations
int semctl(int semid, int semnum, int cmd, ... /* union semun arg */);
union semun { // Must define yourself!
  int val; // for SETVAL
  struct semid_ds *buf; // for IPC_STAT/IPC_SET
  unsigned short *array; // for GETALL/SETALL
};

Commands: SETVAL (set value), GETVAL (get value), SETALL (set all), GETALL (get all), IPC_RMID (delete), IPC_STAT (read info).

Helper Functions Pattern
void sem_P(int semid, int sem_num) { // DOWN struct sembuf sb; sb.sem_num = sem_num; sb.sem_op = -1; // decrement sb.sem_flg = 0; semop(semid, &sb, 1); } void sem_V(int semid, int sem_num) { // UP struct sembuf sb; sb.sem_num = sem_num; sb.sem_op = 1; // increment sb.sem_flg = 0; semop(semid, &sb, 1); } void sem_Init(int semid, int sem_num, int val) { union semun arg; arg.val = val; semctl(semid, sem_num, SETVAL, arg); }
🧠
Tip — union semun
Το union semun δεν ορίζεται από τα headers σε πολλά συστήματα — πρέπει να το ορίσεις ο ίδιος! Κλασική εξεταστική ερώτηση.

9. Semaphore Examples

Server + SHM + Semaphore
#define SHMKEY 1000 #define SEMKEY 2000 int main() { // Create shared memory int shmid = shmget(SHMKEY, 64, IPC_CREAT|0666); char *ptr = (char *)shmat(shmid, NULL, 0); // Create semaphore (init to 0 = locked) int semid = semget(SEMKEY, 1, IPC_CREAT|0666); union semun arg; arg.val = 0; semctl(semid, 0, SETVAL, arg); // Write data strcpy(ptr, "Hello from server"); // Signal client (UP: 0→1) struct sembuf sb = {0, 1, 0}; // sem_op=+1 semop(semid, &sb, 1); printf("Server: data written, signaled\n"); shmdt(ptr); return 0; }
Client — Wait then Read
int main() { int shmid = shmget(SHMKEY, 0, 0); char *ptr = (char *)shmat(shmid, NULL, 0); int semid = semget(SEMKEY, 0, 0); // Wait for server (DOWN: blocks if sem=0) struct sembuf sb = {0, -1, 0}; // sem_op=-1 semop(semid, &sb, 1); printf("Client read: %s\n", ptr); // Cleanup shmdt(ptr); shmctl(shmid, IPC_RMID, NULL); semctl(semid, 0, IPC_RMID); return 0; }
Fork + SHM + Semaphore Counter
int main() { // Shared counter via shared memory int shmid = shmget(IPC_PRIVATE, sizeof(int), IPC_CREAT|0666); int *counter = (int *)shmat(shmid, NULL, 0); *counter = 0; // Semaphore for mutual exclusion (init=1) int semid = semget(IPC_PRIVATE, 1, IPC_CREAT|0666); sem_Init(semid, 0, 1); if (fork() == 0) { // Child: increment 100000 times for (int i = 0; i < 100000; i++) { sem_P(semid, 0); // DOWN (lock) (*counter)++; sem_V(semid, 0); // UP (unlock) } exit(0); } // Parent: same for (int i = 0; i < 100000; i++) { sem_P(semid, 0); (*counter)++; sem_V(semid, 0); } wait(NULL); printf("Counter = %d\n", *counter); // Should be 200000 (correct with semaphore) shmdt(counter); shmctl(shmid, IPC_RMID, NULL); semctl(semid, 0, IPC_RMID); return 0; }
Without semaphore
Χωρίς τα sem_P/sem_V, ο counter θα ήταν λιγότερο από 200000 λόγω race condition!

10. Semaphore Challenges

Τα semaphores λύνουν προβλήματα αλλά δημιουργούν νέα αν δεν χρησιμοποιηθούν σωστά. Τρεις βασικοί κίνδυνοι: Deadlock, Starvation, Priority Inversion.

Deadlock with Semaphores
// Process A // Process B DOWN(S1); DOWN(S2); ... ... DOWN(S2); // WAIT! DOWN(S1); // WAIT! // DEADLOCK: A waits for S2 (held by B) // B waits for S1 (held by A)

Prevention: Πάντα acquire semaphores με σταθερή σειρά (total ordering). Αν S1 < S2 σε αρίθμηση, πάντα DOWN(S1) πριν DOWN(S2).

Starvation

Μια διεργασία δεν παίρνει ποτέ πρόσβαση στον πόρο, ενώ άλλες τρέχουν κανονικά. Συμβαίνει αν ο scheduler ή ο αλγόριθμος δεν είναι δίκαιος (fair). Π.χ. σε Readers/Writers, αν συνεχώς έρχονται νέοι readers, ο writer δεν θα γράψει ποτέ!

Mutual Exclusion Guarantee
// Binary semaphore (init=1) guarantees: // At most ONE process in critical section sem_P(sem, 0); // enter CS /* critical section — only one process here */ sem_V(sem, 0); // leave CS
SEM_UNDO Flag
Αν μια διεργασία πεθάνει κρατώντας ένα semaphore, μπορεί να αφήσει τον πόρο locked (orphaned lock). Η flag SEM_UNDO λέει στον kernel να αναιρέσει αυτόματα τις semaphore operations αν η διεργασία τερματιστεί.
🧠
Exam Tip — Semaphore vs Mutex
Mutex (pthread): Intra-process (threads), ένα thread κλειδώνει/ξεκλειδώνει, lock ordering πρέπει να τηρηθεί.
Semaphore (System V): Inter-process (processes), ένα process μπορεί να κάνει UP χωρίς να έχει κάνει DOWN, counting > 1 possible, SEM_UNDO available.

11. File Locking with fcntl

Η fcntl() επιτρέπει record locking — κλείδωμα τμημάτων ενός αρχείου (byte-range locking). Υποστηρίζει read locks (πολλοί ταυτόχρονα) και write locks (αποκλειστικό). Advisory — δεν εμποδίζει τη φυσική πρόσβαση.

struct flock
struct flock {
  short l_type; // F_RDLCK, F_WRLCK, F_UNLCK
  short l_whence; // SEEK_SET, SEEK_CUR, SEEK_END
  off_t l_start; // starting offset
  off_t l_len; // # bytes (0 = to EOF)
  pid_t l_pid; // PID of blocking process
};
fcntl() Commands for Locking
CommandBehavior
F_GETLKTest if lock is possible. Fills in l_pid of blocker. Δεν κλειδώνει!
F_SETLKSet lock. Returns immediately with error if can't.
F_SETLKWSet lock. Blocks (waits) until lock is acquired.
Lock Types
TypeCompatibility
F_RDLCKRead lock. Multiple readers OK simultaneously.
F_WRLCKWrite lock. Exclusive — blocks all other locks.
F_UNLCKUnlock. Release the byte range.
Usage Pattern
#include <fcntl.h> struct flock my_lock; // Lock first 10 bytes for writing my_lock.l_type = F_WRLCK; my_lock.l_whence = SEEK_SET; my_lock.l_start = 0; my_lock.l_len = 10; int fd = open("datafile", O_RDWR); // Blocking lock (waits if locked) if (fcntl(fd, F_SETLKW, &my_lock) == -1) { perror("lock"); exit(1); } /* ... access bytes 0-9 safely ... */ // Unlock my_lock.l_type = F_UNLCK; fcntl(fd, F_SETLK, &my_lock);
🧠
Tip — F_SETLK vs F_SETLKW
F_SETLK: Non-blocking. Αν δεν μπορεί να κλειδώσει, επιστρέφει αμέσως -1 (errno=EAGAIN). Χρήσιμο για try-lock pattern.
F_SETLKW: Blocking. Περιμένει μέχρι να γίνει δυνατό το lock. Κίνδυνος deadlock!
l_len=0: Κλειδώνει μέχρι το τέλος του αρχείου (ακόμα και bytes που θα γραφτούν αργότερα).

12. File Locking Examples

lockit.c — Parent-Child Lock Example
struct flock my_lock; my_lock.l_type = F_WRLCK; my_lock.l_whence = SEEK_SET; my_lock.l_start = 0; my_lock.l_len = 10; int fd = open("locktest", O_RDWR); fcntl(fd, F_SETLKW, &my_lock); // parent locks bytes 0-9 printf("parent: locked record\n"); switch(fork()) { case 0: // child printf("child: trying to lock file\n"); my_lock.l_len = 5; fcntl(fd, F_SETLKW, &my_lock); // BLOCKS! printf("child: locked\n"); sleep(1); printf("child: exiting\n"); exit(1); default: // parent printf("parent: about to unlock\n"); sleep(5); my_lock.l_type = F_UNLCK; fcntl(fd, F_SETLK, &my_lock); printf("parent: unlocked\n"); wait(NULL); }
Execution Output
parent: locked record
child: trying to lock file
parent: just about unlocking now
parent: unlocking -now-
child: locked
parent: has unlocked and is now exiting
child: exiting
Deadlock with File Locks
// Parent locks bytes 0-9 (first_lock) // Child locks bytes 10-14 (second_lock) // Then... case 0: // child fcntl(fd, F_SETLKW, &second_lock); // OK: got B printf("B: lock obtained\n"); fcntl(fd, F_SETLKW, &first_lock); // BLOCKS: wants A // ... default: // parent printf("Parent sleeping\n"); sleep(10); // Try non-blocking lock if (fcntl(fd, F_SETLK, &second_lock) == -1) { perror("D: Resource temporarily unavailable"); // Parent detects deadlock and exits! }
Deadlock Scenario
Parent κρατά A, child κρατά B. Child θέλει A (F_SETLKW → blocks). Parent θέλει B — αν χρησιμοποιήσει F_SETLK (non-blocking), ανιχνεύει το πρόβλημα και μπορεί να τερματίσει. Αν χρησιμοποιούσε F_SETLKW, θα ήταν deadlock!

Exam Tip — Deadlock Prevention
Χρησιμοποίησε F_SETLK (non-blocking) αντί F_SETLKW για τη δεύτερη κλήση. Αν αποτύχει, ξεκλείδωσε τα πάντα και δοκίμασε ξανά. Αυτό αποτρέπει deadlock αλλά μπορεί να οδηγήσει σε livelock αν δεν σχεδιαστεί σωστά. Εναλλακτικά: κλείδωνε πάντα με σταθερή σειρά (byte ranges σε αύξουσα σειρά).
