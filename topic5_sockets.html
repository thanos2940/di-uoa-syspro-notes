Εισαγωγή στα Sockets & Network IPC

Τα Sockets είναι το de facto standard για την επικοινωνία διεργασιών μέσω δικτύου. Φανταστείτε τα σαν μια "πόρτα" (door) μέσω της οποίας μια διεργασία στέλνει και λαμβάνει μηνύματα.

Communication Models
  • Client-Server: Ο Server είναι πάντα ενεργός (always-on) με σταθερή IP. Ο Client ξεκινά την επικοινωνία.
  • Peer-to-Peer (P2P): Δεν υπάρχει κεντρικός server. Τα συστήματα (peers) επικοινωνούν απευθείας και δρουν και ως clients και ως servers (π.χ. Gnutella).
  • Hybrid: Συνδυασμός και των δύο (π.χ. Skype για εύρεση χρήστη μέσω server αλλά P2P για την κλήση).
AddressingIP & Port
Κάθε συσκευή στο δίκτυο έχει μια IP Address. Κάθε υπηρεσία/διεργασία σε αυτή τη συσκευή έχει ένα Port Number.
  • IPv4: 32-bit (π.χ. 147.52.17.2).
  • IPv6: 128-bit (π.χ. 2001:0db8...).
  • Common Ports: HTTP (80), SSH (22), DNS (53), SMTP (25).
📡
DNS (Domain Name System)
Μεταφράζει ονόματα (π.χ. google.com) σε IP διευθύνσεις, καθώς οι υπολογιστές επικοινωνούν μόνο με αριθμούς.

Τύποι Sockets & Transport Services

Το OS προσφέρει δύο κύριους τύπους transport services μέσω του Socket API:

TCP (SOCK_STREAM)
Reliable Byte Stream: Εγγυάται ότι τα δεδομένα θα φτάσουν σωστά και στη σωστή σειρά.
  • Connection-oriented: Απαιτείται setup (handshake).
  • Flow Control: Ο αποστολέας δεν "πνίγει" τον παραλήπτη.
  • Congestion Control: Μείωση ρυθμού αν το δίκτυο είναι υπερφορτωμένο.
UDP (SOCK_DGRAM)
Unreliable Datagrams: Δεν υπάρχουν εγγυήσεις. Τα δεδομένα μπορεί να χαθούν ή να έρθουν με λάθος σειρά.
  • Connectionless: Δεν υπάρχει setup.
  • Fast: Πολύ χαμηλότερο overhead από το TCP.
  • Χρήση: Streaming video, online games, DNS.
Byte Order (htons/htonl)
Στο δίκτυο χρησιμοποιούμε Network Byte Order (Big Endian). Οι συναρτήσεις htons (host to network short) κ.λπ. είναι απαραίτητες για φορητότητα κώδικα.

Server Architectures & Lifecycle

Server ➔ socket() ➔ bind() ➔ listen() ➔ accept()
Client ➔ socket() ➔ connect()
➔ read() / write() (Data Exchange) ➔ close()
Serial Server
Εξυπηρετεί έναν client τη φορά. Όλοι οι άλλοι περιμένουν στην ουρά.
for(;;) { accept(); handle(); close(); }
Concurrent Server (Fork)
Για κάθε client, ο server κάνει fork() και ορίζει ένα child process για την εξυπηρέτησή του.
newsock = accept();
if(fork() == 0) { child_work(newsock); }
🧩
Γιατί ο Parent κλείνει το newsock;
Μετά το fork(), ο parent πρέπει να κλείσει το newsock. Αν δεν το κάνει, θα ξεμείνει από file descriptors και το socket δεν θα καταστραφεί ποτέ (αφού θα υπάρχουν δύο ανοιχτά refs).

The Socket API & DNS Functions

gethostbyname()
Μετατρέπει ένα hostname (google.com) σε IP διεύθυνση.
struct hostent *h = gethostbyname("google.com");
socket() & bind()
int s = socket(AF_INET, SOCK_STREAM, 0);
bind(s, &addr, sizeof(addr));
listen() & accept()
Ορίζουν το μέγεθος της ουράς (backlog) και δέχονται τη σύνδεση.
listen(sock, 5);
int client = accept(s, NULL, NULL);

SIGPIPE Warning
Αν γράψετε σε ένα socket που έχει κλείσει από την άλλη πλευρά, θα λάβετε το σήμα SIGPIPE. Από προεπιλογή, αυτό τερματίζει τη διεργασία σας!
