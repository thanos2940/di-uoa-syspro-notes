1. Cross-host IPC & Communication Models

Μέχρι τώρα είδαμε IPC στο ίδιο μηχάνημα (pipes, signals, shared memory). Τι γίνεται αν οι διεργασίες τρέχουν σε διαφορετικά μηχανήματα με ένα δίκτυο ανάμεσά τους; Η απάντηση είναι τα Sockets — το βασικό API για δικτυακή επικοινωνία στο UNIX (BSD 4.1, 1981).

Communication Models
  • Client-Server: Ασυμμετρία ρόλων. Ο Server (always-on host, static IP, passive) περιμένει αιτήσεις. Ο Client (intermittent, dynamic IP, active) ξεκινάει τη σύνδεση. Clients δεν επικοινωνούν μεταξύ τους απευθείας.
    Παραδείγματα: Web (HTTP), Email (SMTP), SSH, FTP
  • Peer-to-Peer (P2P): Συμμετρία. Δεν υπάρχει always-on server. Κάθε peer είναι ταυτόχρονα client και server. Highly scalable αλλά δύσκολο στη διαχείριση.
    Παράδειγμα: Gnutella, BitTorrent
  • Hybrid: Server για metadata/discovery, P2P για data transfer. Ο server βρίσκει ποιος είναι online, μετά τα peers μιλάνε απευθείας.
    Παράδειγμα: Skype (centralized lookup, direct client-client voice), Instant Messaging
🧠
Tip — Socket = "πόρτα" (door)
Ένα socket είναι η "πόρτα" μεταξύ application layer και transport layer (TCP/UDP).
Η εφαρμογή ρίχνει δεδομένα από τη μία μεριά — το OS αναλαμβάνει να τα μεταφέρει στο socket του παραλήπτη.
API: (1) Επιλέγεις transport protocol, (2) Δίνεις IP+Port, (3) Στέλνεις/Λαμβάνεις.
Application Protocol

Ορίζει τους κανόνες επικοινωνίας:

  • Τύποι μηνυμάτων: request, response
  • Syntax: ποια πεδία, πώς χωρίζονται
  • Semantics: τι σημαίνει κάθε πεδίο
  • Rules: πότε/πώς στέλνονται
Public vs Proprietary
Public: HTTP, SMTP (ορίζονται σε RFCs, interoperable). Proprietary: Skype, KaZaA (κλειστά πρωτόκολλα).

2. Addressing: IP & Ports

Για να "βρεις" μια διεργασία στο δίκτυο χρειάζεσαι δύο πράγματα: μια IP (ποιο μηχάνημα) και ένα Port (ποια διεργασία σε αυτό το μηχάνημα). Η IP μόνη της δεν αρκεί — πολλές διεργασίες τρέχουν στο ίδιο host.

The IP Layer (L3)
  • IPv4: 32 bits (4 octets). Π.χ. 147.102.12.1 — 4 δεκαδικοί αριθμοί χωρισμένοι με τελεία.
  • IPv6: 128 bits (8 blocks × 16 bits). Π.χ. 2001:0db8:0a0b:12f0::0001
  • Localhost: 127.0.0.1 — "μιλάω στον εαυτό μου", χωρίς να βγει πακέτο στο δίκτυο.
Addressing = IP + Port
Π.χ. για να στείλεις HTTP request στο www.di.uoa.gr: IP = 147.52.17.2, Port = 80.
NAT (Network Address Translation)
Το σπίτι σου έχει μία δημόσια IP (π.χ. 79.166.80.131) από τον ISP. Ο router δίνει εσωτερικές IPs (192.168.x.y) μέσω DHCP. Κρατά πίνακα ποιο πακέτο πάει σε ποιο εσωτερικό PC. Αν θέλεις service εσωτερικά accessible: port forwarding.
The Port Layer (L4)

Ένα 16-bit unsigned integer (0–65535) που δείχνει ποια διεργασία παίρνει το πακέτο.

  • Well-known (0–1023): Δεσμευμένα. HTTP=80, SSH=22, SMTP=25, DNS=53, DHCP=67/68. Χρειάζονται root.
  • Registered (1024–49151): Για εφαρμογές. MySQL=3306, PostgreSQL=5432.
  • Ephemeral (49152–65535): Προσωρινά. Ο client παίρνει αυτόματα ένα port σε αυτό το range.
🧠
Tip — IP vs Port
IP = η διεύθυνση του σπιτιού (ποιο κτίριο).
Port = ο αριθμός διαμερίσματος (ποια πόρτα στο κτίριο).
Χωρίς port, ο postman ξέρει σε ποιο κτίριο να πάει αλλά όχι σε ποιο ένοικο.

3. TCP vs UDP: Transport Services

Το OS παρέχει δύο transport protocols μέσω του Socket API. Ο TCP εγγυάται αξιόπιστη, in-order παράδοση (σαν τηλεφωνική κλήση). Ο UDP στέλνει πακέτα "fire and forget" (σαν ταχυδρομικά γράμματα). Η εφαρμογή αποφασίζει τι χρειάζεται: data loss tolerance, timing, bandwidth.

Χαρακτηριστικό TCP (SOCK_STREAM) UDP (SOCK_DGRAM)
Σύνδεση Connection-oriented (handshake required) Connectionless (no setup)
Αξιοπιστία Εγγυημένη (ACKs, retransmissions) Best-effort (μπορεί να χαθεί!)
Σειρά In-order παράδοση (reorder by OS) Unordered (ό,τι φτάσει)
Message Boundaries Όχι — byte stream, χωρίς όρια Ναι — κάθε datagram αυτόνομο
Flow Control Ναι (sender won't overwhelm receiver) Όχι
Congestion Control Ναι (throttle on overload) Όχι
Socket Type SOCK_STREAM SOCK_DGRAM
Use TCP for:
  • Web (HTTP/HTTPS) — χρειάζεται 100% αξιοπιστία
  • File Transfer (FTP/SCP)
  • Email (SMTP), Remote Login (SSH/Telnet)
  • Database Queries
Use UDP for:
  • Streaming Media — latency > quality
  • Online Gaming — real-time, tolerate loss
  • DNS Queries — single request/response
  • Internet Telephony (VoIP)
🧠
Tip — πότε UDP;
Αν η εφαρμογή μπορεί να ανεχτεί κάποια απώλεια δεδομένων (audio, video) αλλά χρειάζεται χαμηλή καθυστέρηση, τότε UDP. Αν χρειάζεται 100% reliability (αρχεία, email, web), τότε TCP. Ο TCP δεν εγγυάται timing ή bandwidth — μόνο reliability.

4. Socket Fundamentals: Domains, Types & socket()

Ένα socket είναι ένα communication endpoint. Οι διεργασίες αναφέρονται σε αυτό μέσω ενός integer file descriptor — ακριβώς όπως με τα αρχεία. Η socket() δημιουργεί ένα socket και επιστρέφει αυτόν τον fd.

Address Domains (AF_*)

Καθορίζει το format των διευθύνσεων:

  • AF_UNIX / AF_LOCAL: Επικοινωνία εντός ίδιου PC. Η "διεύθυνση" είναι ένα filesystem path. Γρηγορότερο γιατί δεν περνά από network stack.
  • AF_INET: IPv4 Internet domain. Η "διεύθυνση" = 4-byte IP + 2-byte Port. Αυτό θα χρησιμοποιούμε σχεδόν πάντα.
  • AF_INET6: IPv6 (16-byte IP address).
Socket Types (SOCK_*)
  • SOCK_STREAM: TCP. Reliable, in-order byte stream. Σαν σωλήνα νερού — συνεχής ροή.
  • SOCK_DGRAM: UDP. Unreliable, independent datagrams. Σαν ταχυδρομικά γράμματα.
  • SOCK_RAW: Bypass transport layer. Για εργαλεία όπως ping. Χρειάζεται root.
#include <sys/types.h> #include <sys/socket.h> // Prototype: // int socket(int domain, int type, int protocol) // Δημιουργεί socket, επιστρέφει fd ή -1 int sock; if ((sock = socket(AF_INET, SOCK_STREAM, 0)) == -1) perror_exit("socket"); // domain: AF_INET (IPv4), AF_INET6, AF_UNIX // type: SOCK_STREAM (TCP), SOCK_DGRAM (UDP) // protocol: πάντα 0 (default for the type)
🧠
Tip — socket = file descriptor
Το socket είναι fd. Μπορείς να κάνεις read(), write(), close() πάνω του — ακριβώς όπως σε αρχείο ή pipe.
Η μόνη διαφορά: πριν κάνεις read/write, πρέπει πρώτα να συνδέσεις τα δύο endpoints (bind, listen, connect, accept).
Όλα τα socket syscalls
Απαιτούν #include <sys/types.h> και #include <sys/socket.h>. Επιστρέφουν -1 σε αποτυχία (πάντα ελέγχουμε!).

5. Network Byte Order

Το Internet μιλάει Big Endian (Most Significant Byte πρώτο) — αυτό λέγεται Network Byte Order. Οι Intel CPUs μιλάνε Little Endian (LSB πρώτο). Αν δεν κάνουμε μετατροπή, ο αριθμός 0xD04C στον Intel θα διαβαστεί ως 0x4CD0 στο δίκτυο!

#include <sys/types.h> #include <netinet/in.h> // Οι 4 "χρυσές" συναρτήσεις μετατροπής uint16_t htons(host_short); // Host TO Network Short uint32_t htonl(host_long); // Host TO Network Long uint16_t ntohs(net_short); // Network TO Host Short uint32_t ntohl(net_long); // Network TO Host Long // Παράδειγμα: uint16_t n = 0xD04C; uint16_t net = htons(n); // Intel (LE): n bytes = 4C D0 // After htons: net bytes = D0 4C (Network Order) // Σε Big Endian machine: δεν αλλάζει τίποτα!
🧠
Mnemonic — h/n/s/l
h = Host (η δική μου μηχανή).
n = Network (Internet byte order).
s = Short (2 bytes = 16 bits) → για ports.
l = Long (4 bytes = 32 bits) → για IPs.
htons = "Host TO Network Short" = μετατρέπω port πριν στείλω.
ntohl = "Network TO Host Long" = μετατρέπω IP αφού λάβω.
🚨 Πότε μετατρέπουμε
ΠΑΝΤΑ πριν βάλουμε port ή IP στο sockaddr_in. Αλλιώς ο server ακούει σε λάθος port!
srv.sin_port = htons(8080); — ποτέ χωρίς htons.

6. Socket Address Structures (sockaddr_in)

Κάθε socket function (bind, connect, accept) χρειάζεται μια διεύθυνση. Στο Internet domain (AF_INET) χρησιμοποιούμε την struct sockaddr_in, η οποία κάνει cast σε generic struct sockaddr *.

#include <netinet/in.h> struct sockaddr_in { sa_family_t sin_family; /* AF_INET */ in_port_t sin_port; /* port σε NBO */ struct in_addr sin_addr; /* IPv4 address */ }; struct in_addr { in_addr_t s_addr; /* uint32_t σε NBO */ }; /* in_port_t = uint16_t (2 bytes) */ /* in_addr_t = uint32_t (4 bytes) */
Πώς γεμίζουμε τη δομή
struct sockaddr_in server; // 1. Address family: πάντα AF_INET server.sin_family = AF_INET; // 2. Port: ΠΑΝΤΑ htons()! server.sin_port = htons(8080); // 3. IP: INADDR_ANY = όλα τα interfaces server.sin_addr.s_addr = htonl(INADDR_ANY); // Cast σε generic sockaddr* για bind(): bind(sock, (struct sockaddr *)&server, sizeof(server));
🧠
Tip — INADDR_ANY
INADDR_ANY = 0.0.0.0 = "δέξου συνδέσεις σε όλα τα network interfaces του μηχανήματος". Αν ο server έχει πολλές κάρτες δικτύου, δεν χρειάζεται να επιλέξεις μία.
Γιατί (struct sockaddr *) cast;
Η bind() δέχεται struct sockaddr * — μια generic δομή. Αυτό επιτρέπει στο ίδιο API να δουλεύει με AF_INET (sockaddr_in), AF_INET6 (sockaddr_in6), AF_UNIX (sockaddr_un). Εμείς πάντα περνάμε sockaddr_in με cast.

7. DNS Resolution & Address Conversion

Πρέπει να μετατρέψουμε hostnames (π.χ. di.uoa.gr) σε binary IP, και binary IP σε printable string. Δύο κατηγορίες: DNS lookup (hostname ↔ IP) και IP conversion (string ↔ binary).

gethostbyname() — DNS Lookup
#include <netdb.h> struct hostent *h; h = gethostbyname("www.google.com"); if (h == NULL) { herror("gethostbyname"); exit(1); } // h->h_addr_list[0] = IP σε NBO // h->h_name = official hostname // h->h_length = μήκος address (4 για IPv4)
struct hostent { char *h_name; /* official name */ char **h_aliases; /* alt names */ int h_addrtype; /* AF_INET */ int h_length; /* addr len (4) */ char **h_addr_list; /* IP array */ };
IP Conversion Functions
#include <arpa/inet.h> // OLD: μόνο IPv4 // string "1.2.3.4" → binary (NBO) inet_aton("1.2.3.4", &addr); // returns 1/0 // binary (NBO) → string "1.2.3.4" char *s = inet_ntoa(addr); // static buffer! // NEW: IPv4 + IPv6 (POSIX.1-2001) // presentation → network (pton = "to network") inet_pton(AF_INET, "1.2.3.4", &addr); // network → presentation (ntop = "to presentation") inet_ntop(AF_INET, &addr, buf, sizeof(buf));
🚨 Deprecation
gethostbyname() είναι NOT thread-safe (static buffer). POSIX.1-2001 συστήνει getaddrinfo() / getnameinfo(). Ομοίως inet_ntoa() χρησιμοποιεί static buffer — prefer inet_ntop().
🧠
Mnemonic — aton/ntoa/pton/ntop
aton = ASCII "to" network.
ntoa = network "to" ASCII.
pton = presentation "to" network.
ntop = network "to" presentation.
Τα "new" (pton/ntop) δουλεύουν και με IPv6.
Πλήρες παράδειγμα: hostname → IP string
struct hostent *mymachine; struct in_addr **addr_list; char symbolicip[50]; if ((mymachine = gethostbyname(hostname)) == NULL) { herror("gethostbyname"); exit(1); } addr_list = (struct in_addr **) mymachine->h_addr_list; for (int i = 0; addr_list[i] != NULL; i++) { strcpy(symbolicip, inet_ntoa(*addr_list[i])); printf("%s resolved to %s\n", mymachine->h_name, symbolicip); } // Output: di.uoa.gr resolved to 195.134.65.123
🧠
Exam Tip — Thread-Safety & DNS / Address Conversion
Στις εξετάσεις ζητείται συχνά η εξήγηση γιατί οι κλασικές συναρτήσεις δικτύου δεν είναι thread-safe:
  • gethostbyname() & inet_ntoa(): Επιστρέφουν pointer σε έναν κοινόχρηστο static buffer που διαχειρίζεται το OS. Αν δύο threads τις καλέσουν ταυτόχρονα, τα δεδομένα του ενός θα υπεργράψουν του άλλου (data corruption).
  • getaddrinfo() & inet_ntop(): Είναι οι σύγχρονες reentrant εναλλακτικές. Η getaddrinfo() δεσμεύει δυναμικά μνήμη (heap) ανά κλήση (πρέπει να την ελευθερώσεις με freeaddrinfo()), ενώ η inet_ntop() απαιτεί από τον caller να περάσει δικό του buffer (caller-owned memory), εξαλείφοντας το κοινόχρηστο state.

8. TCP Server Lifecycle — Πλήρης Ανάλυση

Η ζωή ενός TCP server ακολουθεί αυστηρή σειρά: socket() → bind() → listen() → accept() → read()/write() → close(). Κάθε βήμα είναι κρίσιμο — αν παραλείψεις ένα, ο server δεν λειτουργεί.

listen()

accept()
Αναμονή για Client
→
read() / write()
Ανταλλαγή δεδομένων
→

Κλείσιμο σύνδεσης
bind() — Κατοχύρωση Port
// int bind(int socket, // const struct sockaddr *address, // socklen_t address_len); struct sockaddr_in server; server.sin_family = AF_INET; server.sin_addr.s_addr = htonl(INADDR_ANY); server.sin_port = htons(port); if (bind(sock, (struct sockaddr *)&server, sizeof(server)) < 0) perror_exit("bind");

Χωρίς bind, το OS δεν ξέρει σε ποιο port να στείλει τα εισερχόμενα πακέτα στον server.

listen() & accept()
// int listen(int socket, int backlog); // Μέχρι backlog connections στην ουρά if (listen(sock, 5) < 0) perror_exit("listen"); // int accept(int socket, // struct sockaddr *address, // socklen_t *address_len); socklen_t clientlen = sizeof(client); cfd = accept(sock, (struct sockaddr *)&client, &clientlen); // BLOCKS μέχρι client connects! // Επιστρέφει ΝΕΟ fd για τη σύνδεση // client struct γεμίζει με τα στοιχεία του
The Two-Socket Model
lfd (Listening Socket) = η ρεσεψιόν του ξενοδοχείου. Δέχεται αιτήσεις, δεν εξυπηρετεί κανέναν.
cfd (Connected Socket) = το κλειδί δωματίου. Ξεχωριστό για κάθε client. Πάνω σε αυτό γίνεται η read/write.

Η accept() αφήνει τον lfd ανέπαφο και δημιουργεί ΝΕΟ cfd. Έτσι ο server μπορεί να εξυπηρετεί πολλούς clients — κάθε accept() δίνει νέο cfd.
🧠
Tip — backlog parameter
Ο αριθμός στο listen(sock, 5) = πόσες ολοκληρωμένες αλλά μη-αποδεκτές συνδέσεις κρατάει ο kernel στην ουρά. Αν γεμίσει, νέα connections αρνούνται. Τυπικά: 5–128.
🧠
Exam Tip — accept() & Value-Result Parameters
Η παράμετρος clientlen στην accept() είναι κλασικό παράδειγμα Value-Result Parameter:
  • Value (Είσοδος): Πριν την κλήση, πρέπει να θέσεις clientlen = sizeof(client). Αυτό ενημερώνει τον kernel για το μέγιστο διαθέσιμο μέγεθος του buffer (π.χ. size of struct sockaddr_in).
  • Result (Έξοδος): Όταν η accept() επιστρέψει, ο kernel γράφει στο clientlen το πραγματικό μέγεθος της διεύθυνσης του client που αποθηκεύτηκε.
🚨 Κίνδυνος: Αν ξεχάσεις να αρχικοποιήσεις το clientlen, η accept() θα διαβάσει τυχαία τιμή από τη μνήμη (σκουπίδια). Αν αυτή η τιμή τύχει να είναι πολύ μικρή (π.χ. 0), η κλήση θα αποτύχει ή θα επιστρέψει ατελή/κατεστραμμένα στοιχεία!
Πλήρης Annotated TCP Server (inet_str_server)
#include <stdio.h> #include <sys/wait.h> /* wait */ #include <sys/types.h> /* sockets */ #include <sys/socket.h> /* sockets */ #include <netinet/in.h> /* internet sockets */ #include <netdb.h> /* gethostbyaddr */ #include <unistd.h> /* fork, close */ #include <stdlib.h> /* exit */ #include <ctype.h> /* toupper */ #include <signal.h> /* signal */ void sigchld_handler(int sig) { while (waitpid(-1, NULL, WNOHANG) > 0); } void child_server(int newsock) { char buf[1]; while (read(newsock, buf, 1) > 0) { /* Receive 1 char */ putchar(buf[0]); buf[0] = toupper(buf[0]); /* Capitalize */ if (write(newsock, buf, 1) < 0) perror_exit("write"); /* Reply */ } printf("Closing connection.\n"); close(newsock); } int main(int argc, char *argv[]) { int port, sock, newsock; struct sockaddr_in server, client; socklen_t clientlen; struct hostent *rem; port = atoi(argv[1]); signal(SIGCHLD, sigchld_handler); /* Reap zombies */ sock = socket(AF_INET, SOCK_STREAM, 0); /* 1. Create socket */ server.sin_family = AF_INET; server.sin_addr.s_addr = htonl(INADDR_ANY); server.sin_port = htons(port); bind(sock, (struct sockaddr*)&server, /* 2. Bind */ sizeof(server)); listen(sock, 5); /* 3. Listen */ while (1) { clientlen = sizeof(client); newsock = accept(sock, /* 4. Accept (BLOCKS) */ (struct sockaddr*)&client, &clientlen); rem = gethostbyaddr( /* Who connected? */ (char*)&client.sin_addr.s_addr, sizeof(client.sin_addr.s_addr), client.sin_family); printf("Accepted connection from %s\n", rem->h_name); switch (fork()) { /* 5. Fork child */ case -1: perror("fork"); break; case 0: /* CHILD */ close(sock); /* Close listener! */ child_server(newsock); /* Handle client */ exit(0); } close(newsock); /* PARENT: close client fd immediately */ } }

9. TCP Client — Πλήρης Ανάλυση

Ο client είναι πιο απλός: socket() → connect() → read()/write() → close(). Δεν χρειάζεται bind, listen ή accept — απλά συνδέεται στον server και αρχίζει να μιλάει.

Τερματισμός
connect() — Σύνδεση στον Server
// int connect(int socket, // struct sockaddr *address, socklen_t address_len); struct sockaddr_in server; struct hostent *rem; rem = gethostbyname(hostname); /* Resolve server IP */ server.sin_family = AF_INET; memcpy(&server.sin_addr, rem->h_addr, rem->h_length); server.sin_port = htons(port); if (connect(sock, (struct sockaddr*)&server, sizeof(server)) < 0) perror_exit("connect"); // TCP handshake ολοκληρώνεται — μπορούμε read/write!
Πλήρης Annotated TCP Client (inet_str_client)
#include <stdio.h> #include <sys/types.h> #include <sys/socket.h> #include <netinet/in.h> #include <unistd.h> /* read, write, close */ #include <netdb.h> #include <stdlib.h> #include <string.h> int main(int argc, char *argv[]) { int port, sock, i; char buf[256]; struct sockaddr_in server; struct hostent *rem; /* 1. Create socket */ if ((sock = socket(AF_INET, SOCK_STREAM, 0)) < 0) perror_exit("socket"); /* 2. Resolve server & connect */ if ((rem = gethostbyname(argv[1])) == NULL) { herror("gethostbyname"); exit(1); } port = atoi(argv[2]); server.sin_family = AF_INET; memcpy(&server.sin_addr, rem->h_addr, rem->h_length); server.sin_port = htons(port); if (connect(sock, (struct sockaddr*)&server, sizeof(server)) < 0) perror_exit("connect"); printf("Connecting to %s port %d\n", argv[1], port); /* 3. Send/Receive loop */ do { printf("Give input string: "); fgets(buf, sizeof(buf), stdin); for (i = 0; buf[i] != '\0'; i++) { if (write(sock, buf + i, 1) < 0) /* Send char */ perror_exit("write"); if (read(sock, buf + i, 1) < 0) /* Recv reply */ perror_exit("read"); } printf("Received string: %s", buf); } while (strcmp(buf, "END\n") != 0); close(sock); /* 4. Close */ }
Client δεν χρειάζεται bind()
Ο client δεν κάνει bind — το OS αυτόματα αναθέτει ένα ephemeral port (49152–65535). Ο server ξεχωρίζει τους clients από τα source ports.
shutdown() vs close()
shutdown(fd, SHUT_WR) = "τελείωσα να στέλνω, αλλά μπορώ ακόμα να λαμβάνω". Χρήσιμο σε HTTP: ο client στέλνει request → shutdown(SHUT_WR) → ο server βλέπει EOF → στέλνει response → close(). Options: SHUT_RD, SHUT_WR, SHUT_RDWR.

10. Server Architectures

Πώς διαχειριζόμαστε πολλούς clients ταυτόχρονα; Υπάρχουν τρεις βασικές αρχιτεκτονικές, η καθεμία με trade-offs.

1. Iterative (Serial) Server
for (;;) { cfd = accept(lfd, ...); /* Wait for client */ while (no error) read/write(cfd, ...); /* Serve ONE client */ close(cfd); }
  • Ελάχιστη μνήμη, μηδενικό overhead
  • Αν ο Client 1 αργήσει, ο Client 2 κάνει timeout!
  • Μόνο για πολύ γρήγορα requests (Time server)
2. Concurrent (Fork) Server
for (;;) { cfd = accept(lfd, ...); if (fork() == 0) { /* CHILD */ close(lfd); /* Δεν χρειάζεται listener */ handle_request(cfd); close(cfd); exit(0); } close(cfd); /* PARENT: κλείνει cfd ΑΜΕΣΩΣ */ }
  • Πλήρης απομόνωση μεταξύ clients
  • Απλό coding (κλασικό UNIX)
  • fork() = ακριβό (copy memory)
  • Δύσκολη inter-child επικοινωνία
🚨 Γιατί ο parent κλείνει cfd;
Τρεις λόγοι: (1) Αποφεύγει εξάντληση file descriptors. (2) Μόνο όταν ΟΛΑ τα references στο socket κλείσουν (child + client) γίνεται πραγματικός τερματισμός — αν ο parent κρατά cfd ανοιχτό, δεν κλείνει ποτέ. (3) Το child δεν θα δει EOF αν ο client κλείσει, γιατί ο parent ακόμα "κρατάει" το socket.
SIGCHLD Handler — Zombie Prevention
void sigchld_handler(int sig) { while (waitpid(-1, NULL, WNOHANG) > 0); /* Loop: μάζεψε ΟΛΑ τα τελειωμένα children */ } // Εγκατάσταση πριν το main loop: signal(SIGCHLD, sigchld_handler);
🧠
Tip — γιατί while + WNOHANG;
Αν πολλά children τελειώσουν ταυτόχρονα, μπορεί να λάβεις ένα μόνο SIGCHLD (signals δεν κάνουν queue). Χωρίς while loop θα μαζέψεις μόνο ένα zombie — τα υπόλοιπα μένουν. Το WNOHANG κάνει τον waitpid non-blocking.
🔄

11. I/O Multiplexing με select()

Τρίτη αρχιτεκτονική: Αντί fork() per client, ένα μόνο process παρακολουθεί πολλά sockets ταυτόχρονα μέσω select(). Η select() μπλοκάρει μέχρι κάποιο fd να γίνει "ready" (δεδομένα ήρθαν, μπορείς να γράψεις, κτλ.).

select() — Prototype & Logic
#include <sys/time.h> #include <sys/types.h> int select( int nfds, /* highest fd + 1 */ fd_set *readfds, /* watch for readable */ fd_set *writefds, /* watch for writable */ fd_set *exceptfds, /* watch for exceptions */ struct timeval *timeout ); // Returns: # ready fds, 0=timeout, -1=error
fd_set Macros
fd_set readset; FD_ZERO(&readset); /* Καθάρισε το set */ FD_SET(sock, &readset); /* Πρόσθεσε fd */ FD_CLR(sock, &readset); /* Αφαίρεσε fd */ FD_ISSET(sock, &readset);/* Είναι ready; */
🧠
Tip — Πώς δουλεύει η select()
1. FD_ZERO: αδειάζεις το set.
2. FD_SET: βάζεις όλα τα fds που σε ενδιαφέρουν (listening + connected sockets).
3. select() μπλοκάρει μέχρι κάποιο fd γίνει ready.
4. Μετά την select, η fd_set τροποποιείται: περιέχει μόνο τα ready fds.
5. FD_ISSET: ελέγχεις ποιο fd ήταν η αιτία.
6. Repeat! (πρέπει να ξανακάνεις FD_ZERO + FD_SET κάθε iteration)
Listening socket στο readset
Αν ο listening socket γίνει "readable" στην select(), σημαίνει ότι ένας client κάνει connect(). Τότε κάνουμε accept() και προσθέτουμε τον νέο cfd στο set.
Πλεονέκτημα select()
Ένα μόνο process, χωρίς fork(). Κανένα overhead δημιουργίας processes. Ιδανικό για πολλούς clients με μικρό traffic.
select() Server — Simplified Flow
int highsock; int connectlist[5]; /* Array of connected client fds */ fd_set socks; /* Setup: socket, bind, listen, setnonblocking */ highsock = sock; memset(connectlist, 0, sizeof(connectlist)); while (1) { FD_ZERO(&socks); /* Reset every iteration! */ FD_SET(sock, &socks); /* Add listening socket */ for (int i = 0; i < 5; i++) /* Add all connected clients */ if (connectlist[i] != 0) { FD_SET(connectlist[i], &socks); if (connectlist[i] > highsock) highsock = connectlist[i]; } /* BLOCK until something happens */ int ready = select(highsock + 1, &socks, NULL, NULL, &timeout); /* New connection? */ if (FD_ISSET(sock, &socks)) handle_new_connection(); /* accept() + add to list */ /* Data from existing client? */ for (int i = 0; i < 5; i++) if (FD_ISSET(connectlist[i], &socks)) deal_with_data(i); /* read, process, respond */ }
🚨 nfds = highest fd + 1
Το πρώτο argument της select() δεν είναι ο αριθμός fds — είναι η μεγαλύτερη τιμή fd + 1. Αν έχεις fds 3, 5, 7, περνάς 8. Αυτό λέει στον kernel πόσα bits να ελέγξει στο bitmap.
⚡

12. UDP API: sendto() & recvfrom()

Στο UDP δεν υπάρχει connection — σκέψου ταχυδρομικά γράμματα, όχι τηλέφωνο. Δεν χρειάζεται listen() ή accept(). Αντί για read/write χρησιμοποιούμε sendto() και recvfrom() που περνάνε τη διεύθυνση σε κάθε πακέτο.

UDP Lifecycle Flow
UDP Server
1. socket(AF_INET, SOCK_DGRAM, 0)
↓
2. bind(port)
↓
3. recvfrom() ↔ sendto() loop
UDP Client
1. socket(AF_INET, SOCK_DGRAM, 0)
↓
2. [bind()] (optional)
↓
3. sendto() ↔ recvfrom() loop
sendto() & recvfrom()
ssize_t sendto( int sock, const void *buf, size_t count, int flags, const struct sockaddr *dest, socklen_t len ); // Σαν write() αλλά ορίζει destination ssize_t recvfrom( int sock, void *buf, size_t count, int flags, struct sockaddr *addr, socklen_t *len ); // Σαν read() αλλά γεμίζει source address // addr/len μπορούν να είναι NULL // flags: συνήθως 0
🧠
Tip — UDP vs TCP API
TCP: socket → bind → listen → accept → read/write → close
UDP: socket → bind → recvfrom/sendto → close
Δεν υπάρχει listen(), accept(), connect() στον UDP server. Κάθε recvfrom() γυρίζει τη διεύθυνση αποστολέα — έτσι ξέρεις σε ποιον να απαντήσεις.
UDP port = 0
Αν κάνεις sin_port = htons(0), το OS αυτόματα επιλέγει ένα ελεύθερο port. Χρησιμοποίησε getsockname() μετά το bind για να μάθεις ποιο πήρες.
UDP Server
sock = socket(AF_INET, SOCK_DGRAM, 0); server.sin_family = AF_INET; server.sin_addr.s_addr = htonl(INADDR_ANY); server.sin_port = htons(0); /* Auto-select */ bind(sock, serverptr, sizeof(server)); getsockname(sock, serverptr, &serverlen); printf("Port: %d\n", ntohs(server.sin_port)); while (1) { clientlen = sizeof(client); n = recvfrom(sock, buf, sizeof(buf), 0, clientptr, &clientlen); sendto(sock, buf, n, 0, clientptr, clientlen); /* Echo back */ }
UDP Client
sock = socket(AF_INET, SOCK_DGRAM, 0); rem = gethostbyname(argv[1]); server.sin_family = AF_INET; memcpy(&server.sin_addr, rem->h_addr, rem->h_length); server.sin_port = htons(atoi(argv[2])); client.sin_family = AF_INET; client.sin_addr.s_addr = htonl(INADDR_ANY); client.sin_port = htons(0); /* Auto port */ bind(sock, clientptr, sizeof(client)); while (fgets(buf, sizeof(buf), stdin)) { sendto(sock, buf, strlen(buf)+1, 0, serverptr, serverlen); recvfrom(sock, buf, sizeof(buf), 0, NULL, NULL); /* Don't care src */ printf("%s\n", buf); }
🚨 UDP είναι unreliable!
Αν ο server αργήσει ή πακέτο χαθεί, ο client κολλάει στο recvfrom() για πάντα. Λύσεις: (1) Timeout μέσω select() πριν το recvfrom, (2) Retransmission logic στην εφαρμογή, (3) Αν χρειάζεται reliability → χρησιμοποίησε TCP!
🚀

13. Professional Pitfalls & Common Bugs

Αυτά τα θέματα εμφανίζονται συνέχεια σε εξετάσεις και σε real-world bugs. Μάθε τα μια φορά, σώσε ώρες debugging.

1. "Address Already in Use"

Σταματάς τον server, τον ξανατρέχεις: bind(): Address Already in Use. Γιατί; Ο kernel κρατάει το port σε TIME_WAIT state μετά το close (60-120 sec).

/* Η ΛΥΣΗ: πριν το bind() */ int val = 1; setsockopt(sock, SOL_SOCKET, SO_REUSEADDR, &val, sizeof(val)); /* Τώρα μπορείς να κάνεις bind ξανά αμέσως */
🧠
Exam Tip — TIME_WAIT & SO_REUSEADDR
Όταν ένας server τερματίζει, το port παραμένει δεσμευμένο από τον kernel σε κατάσταση TIME_WAIT για 1-2 λεπτά (για να ληφθούν τυχόν καθυστερημένα πακέτα). Αν τον ξανατρέξεις αμέσως, η bind() θα αποτύχει με Address already in use.
Η λύση: Ενεργοποιούμε την επαναχρησιμοποίηση της διεύθυνσης πριν τη bind():
  • int val = 1;
  • setsockopt(sock, SOL_SOCKET, SO_REUSEADDR, &val, sizeof(val));
Στις εξετάσεις, η μη χρήση της SO_REUSEADDR θεωρείται σημαντική παράλειψη!
2. SIGPIPE — Silent Server Death

Ο client κλείνει. Εσύ κάνεις write() στο κλειστό socket. Το OS στέλνει SIGPIPE → ο server πεθαίνει σιωπηλά!

Before

write() σε closed socket → SIGPIPE → server killed, χωρίς μήνυμα λάθους.

After
signal(SIGPIPE, SIG_IGN);

write() επιστρέφει -1, errno=EPIPE. Ο server μένει όρθιος!

3. Partial Reads & Writes

Στο TCP, αν στείλεις 100 bytes, η read() μπορεί να επιστρέψει 40 τώρα και 60 αργότερα. Το TCP είναι byte stream, όχι πακέτα. Ποτέ μην υποθέσεις ότι ένα read() διαβάζει τα πάντα.

/* write_all: γράψε μέχρι τελειώσουν ΟΛΑ */ int write_all(int fd, void *buf, size_t size) { int sent, n; for (sent = 0; sent < size; sent += n) { if ((n = write(fd, buf+sent, size-sent)) == -1) return -1; } return sent; }
Golden Rule
ΠΑΝΤΑ loop γύρω από read/write μέχρι τα αναμενόμενα bytes μεταφερθούν (ή error/EOF).
4. Useful Functions
/* Βρες τη διεύθυνση ΤΟΥ socket σου */ getsockname(sock, &addr, &addrlen); /* Βρες τη διεύθυνση ΤΟΥ peer (remote) */ getpeername(sock, &addr, &addrlen); /* Half-close: τέλος αποστολής αλλά */ /* μπορώ ακόμα να λαμβάνω */ shutdown(fd, SHUT_WR); /* Options: SHUT_RD, SHUT_WR, SHUT_RDWR */
netstat
netstat -n -l -p: δείξε listening sockets, numeric, με process info. -t/-u = TCP/UDP only. -a = all states (LISTEN, ESTABLISHED, TIME_WAIT).

14. Real-world Examples

Δύο πρακτικά παραδείγματα από τα slides που δείχνουν πώς συνδυάζονται όλα: sockets, fork, pipes, DNS resolution.

rlsd / rls — Remote ls

Server (rlsd): Δέχεται TCP connection, διαβάζει directory name, τρέχει ls μέσω popen(), στέλνει αποτέλεσμα στον client.

/* rlsd: key logic */ csock = accept(lsock, NULL, NULL); sock_fp = fdopen(csock, "r+"); fgets(dirname, BUFSIZ, sock_fp); sanitize(dirname); /* ← SECURITY! */ snprintf(command, BUFSIZ, "ls %s", dirname); pipe_fp = popen(command, "r"); while ((c = getc(pipe_fp)) != EOF) putc(c, sock_fp); /* pipe → socket */ pclose(pipe_fp); fclose(sock_fp);

Client (rls): connect → send directory name → read response → stdout. Χρησιμοποιεί write_all() pattern.

🚨 Security — sanitize()
Αν ο client στείλει "; rm *" ως dirname, η popen("ls ; rm *") θα σβήσει τα πάντα! Η sanitize() κρατάει μόνο alphanumerics και slashes.
PRS Game — Paper Rock Scissors

Συνδυάζει pipes και sockets σε ένα πρόγραμμα. Τρεις διεργασίες: Referee, Player 1 (local), Player 2 (remote).

  • Referee (prsref): Parent process. Μιλάει στον Player 1 μέσω pipes (fork) και στον Player 2 μέσω socket (TCP accept).
  • Player 1: Child process του referee. Επικοινωνεί μέσω fd1 (parent→child) και fd2 (child→parent).
  • Player 2: Remote process. Κάνει connect() στον referee. Στέλνει/λαμβάνει μέσω socket.
Protocol Design
Τα μηνύματα χρησιμοποιούν length-prefixed format: πρώτα 1 byte (μήκος string), μετά τα bytes. Αυτό λύνει το πρόβλημα "πότε τελείωσε το μήνυμα" σε TCP streams.
🧠
Exam Tip — Pipes + Sockets
Αυτό είναι κλασικό θέμα εξέτασης: ένα πρόγραμμα που χρησιμοποιεί και pipes και sockets — ο ένας παίκτης είναι τοπικός (pipe) και ο άλλος απομακρυσμένος (socket). Βεβαιώσου ότι κλείνεις τα σωστά ends σε κάθε process!
