Εισαγωγή στα Sockets & Network IPC

Τα Sockets είναι το de facto standard για την επικοινωνία διεργασιών μέσω δικτύου. Φανταστείτε τα σαν μια "πόρτα" (door) μέσω της οποίας μια διεργασία στέλνει και λαμβάνει μηνύματα.

Communication Models
  • Client-Server: Ο Server είναι πάντα ενεργός (always-on), έχει σταθερή IP και περιμένει (listen). Ο Client ξεκινά την επικοινωνία.
  • Peer-to-Peer (P2P): Δεν υπάρχει κεντρικός server. Κάθε κόμβος (peer) δρα ταυτόχρονα ως client και ως server (π.χ. Gnutella).
  • Hybrid: Συνδυασμός των δύο (π.χ. Skype για εύρεση χρήστη μέσω κεντρικού server, αλλά direct P2P σύνδεση για την κλήση).
AddressingIP & Port
Για την πρόσβαση σε μια υπηρεσία απαιτείται το ζεύγος (IP, Port).
  • IP Address: Προσδιορίζει το μηχάνημα στο δίκτυο (IPv4: 32-bit, IPv6: 128-bit).
  • Port Number: Προσδιορίζει τη συγκεκριμένη διεργασία/υπηρεσία εντός του host (π.χ. HTTP: 80, SSH: 22).
📡
DNS & NAT
Το DNS μεταφράζει ονόματα σε IPs. Το NAT επιτρέπει σε πολλές εσωτερικές συσκευές να μοιράζονται μία δημόσια IP.

Τύποι Sockets & Transport Services

Το OS προσφέρει δύο κύριους τύπους υπηρεσιών μεταφοράς μέσω του Socket API:

TCP (SOCK_STREAM)
Reliable Byte Stream: Αξιόπιστη, in-order παράδοση δεδομένων (σαν "pipe").
  • Connection-oriented: Απαιτείται εγκατάσταση σύνδεσης (handshake).
  • Flow/Congestion Control: Διαχείριση ταχύτητας για αποφυγή υπερφόρτωσης.
UDP (SOCK_DGRAM)
Unreliable Datagrams: Γρήγορη αποστολή χωρίς εγγυήσεις.
  • Connectionless: Δεν απαιτείται setup.
  • Χρήση: Streaming, VoIP, DNS όπου το latency είναι κρίσιμο.
Network Byte Order
Τα δίκτυα χρησιμοποιούν Big Endian. Οι συναρτήσεις htons() κ.λπ. είναι απαραίτητες για μετατροπή από host order.

Server Architectures & Lifecycle

Server ➔ socket() ➔ bind() ➔ listen() ➔ accept()
Client ➔ socket() ➔ connect()
Serial Server
Εξυπηρετεί έναν client τη φορά. Οι άλλοι περιμένουν.
for(;;) { newsock = accept(); serve(); close(); }
Concurrent (Fork)
Fork ανά σύνδεση. Ο parent δέχεται, το child εξυπηρετεί.
newsock = accept();
if(fork() == 0) { serve(newsock); exit(); }
close(newsock);
Multiplexing (select)
Ένα process διαχειρίζεται πολλά sockets ταυτόχρονα παρακολουθώντας events.
🧩
File Descriptor Cleanup
Μετά το fork(), ο parent πρέπει να κλείσει το newsock για αποφυγή διαρροής FDs.

The Socket API Essentials

socket() & bind()
int s = socket(AF_INET, SOCK_STREAM, 0);
bind(s, &addr, sizeof(addr));
listen() & accept()
listen(s, 5);
int cl = accept(s, NULL, NULL);
gethostbyname()
Επίλυση ονόματος (Legacy API).
struct hostent *h = gethostbyname("di.uoa.gr");

SIGPIPE Handling
Γράψιμο σε κλειστό socket προκαλεί SIGPIPE. Απαιτείται signal handler για αποφυγή τερματισμού.
