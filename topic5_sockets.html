Εισαγωγή στα Sockets

Τα Sockets είναι το de facto standard για την επικοινωνία διεργασιών μέσω δικτύου. Φανταστείτε τα σαν μια "πόρτα" (door) μέσω της οποίας μια διεργασία στέλνει και λαμβάνει μηνύματα.

AddressingIP & Port
Για να βρεθεί μια διεργασία στο δίκτυο, απαιτείται ο συνδυασμός IP Address (ποιο μηχάνημα;) και Port Number (ποια διεργασία;).
  • IP: 32-bit (IPv4) ή 128-bit (IPv6) αναγνωριστικό host.
  • Port: 16-bit αριθμός (0-65535).
  • Συνδυασμός (IP, Port) = Μοναδικό Endpoint.
📡
Client-Server Model
Server: Πάντα ενεργός, σταθερή IP, περιμένει (listen) για συνδέσεις.
Client: Ξεκινά την επικοινωνία, συνδέεται στον server, μπορεί να έχει δυναμική IP.
🔌

Τύποι Sockets & Transport Services

Το OS προσφέρει δύο κύριους τύπους transport services μέσω του Socket API:

TCPSOCK_STREAM
Connection-oriented, αξιόπιστη μεταφορά (byte stream), flow & congestion control.
  • HTTP, FTP, SSH, SMTP.
  • Εγγυάται τη σειρά και την παράδοση των δεδομένων.
UDPSOCK_DGRAM
Unreliable, connectionless, χωρίς εγγυήσεις παράδοσης ή σειράς.
  • Streaming, DNS, VoIP.
  • Χαμηλότερο overhead (latency) - ιδανικό για real-time.
Network Byte Order
Επειδή διαφορετικά CPU έχουν διαφορετικό Endianness (Little vs Big Endian), στο δίκτυο χρησιμοποιούμε πάντα το Big Endian.
htons, htonl, ntohs, ntohl
💡
Little Endian vs Network Order
Ένας x86 (Intel) host είναι little-endian. Πρέπει να μετατρέπουμε τους αριθμούς σε network order πριν τους στείλουμε στο socket!
🔄

Connection Lifecycle (TCP)

Server ➔ socket() ➔ bind() ➔ listen() ➔ accept()
Client ➔ socket() ➔ connect()
➔ read() / write() (Data Exchange) ➔ close()

Ο Server πρέπει να είναι ήδη σε κατάσταση listening πριν ο client επιχειρήσει να συνδεθεί. Η accept() μπλοκάρει μέχρι να έρθει μια νέα σύνδεση.

Ο Client δημιουργεί ένα δικό του socket και καλεί την connect(), η οποία ξεκινά το TCP 3-way handshake.

🛠️

The Socket API

socket()
Δημιουργεί ένα endpoint επικοινωνίας.
int sock = socket(AF_INET, SOCK_STREAM, 0);
bind()
Αναθέτει μια IP και ένα Port στο socket.
bind(sock, (struct sockaddr *)&server, sizeof(server));
listen() & accept()
Προετοιμασία για υποδοχή συνδέσεων και αποδοχή τους.
listen(sock, 5);
int newsock = accept(sock, NULL, NULL);
⚠️
Zombie Clients
Σε concurrent servers, ο parent πρέπει να κάνει wait() (συνήθως μέσω SIGCHLD handler) για να μην μένουν "ζόμπι" τα child processes.
