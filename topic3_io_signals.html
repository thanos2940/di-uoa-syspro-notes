Error Handling — Ο Χρυσός Κανόνας

Κάθε system call επιστρέφει -1 σε αποτυχία και θέτει τη global μεταβλητή errno με τον κωδικό σφάλματος. Ποτέ μην αγνοείς την επιστρεφόμενη τιμή.

// το ΣΩΣΤΟ pattern — πάντα έτσι int fd; if ((fd = open("file.txt", O_RDONLY)) == -1) { perror("open"); // ↑ εκτυπώνει: "open: No such file or directory" exit(1); } // ή με strerror για custom μήνυμα if (write(fd, buf, n) == -1) { fprintf(stderr, "write: %s\n", strerror(errno)); exit(1); }
⚠️ Headers
#include <errno.h> για errno · #include <string.h> για strerror() · #include <stdio.h> για perror()
🧠
Tip — perror vs strerror
perror("msg") = τυπώνει αυτόματα στο stderr: "msg: [error description]". Απλό και βολικό.
strerror(errno) = επιστρέφει string με description. Χρήσιμο αν θέλεις να το ενσωματώσεις σε δικό σου μήνυμα.
🚨 Κρίσιμο
Το errno ορίζεται μόνο αν η κλήση απέτυχε. Αν η κλήση πετύχει, η τιμή errno είναι undefined. Έλεγξε πάντα αν η κλήση απέτυχε πριν κοιτάξεις το errno.
📋

File Descriptors — Ο Πίνακας FD

Φαντάσου ότι κάθε διεργασία κρατά ένα "τηλεφωνικό κατάλογο" ανοιχτών πόρων: αυτό είναι ο πίνακας File Descriptors (FD). Κάθε εγγραφή έχει ένα νούμερο (0, 1, 2...) και δείχνει σε ένα αρχείο, pipe ή socket. Αντί για το όνομα, χρησιμοποιούμε πάντα αυτόν τον αριθμό. Τα πρώτα 3 υπάρχουν ήδη από τη στιγμή που ξεκινά η διεργασία:

PROCESS FD TABLE
0
→stdin — keyboard (terminal)
1
→stdout — screen (terminal)
2
→stderr — screen (terminal)
3
→1ο open() που κάνουμε
4
→2ο open(), κ.ο.κ.
🧠
Tip — γιατί 0, 1, 2
Είναι τα standard streams που κάθε UNIX διεργασία αποκτά αυτόματα όταν ξεκινά. Αντιστοιχούν στα stdin, stdout, stderr της C (αλλά εκεί είναι pointers, εδώ integers).
Κληρονομιά μετά από fork()
Το child process παίρνει αντίγραφο του FD table. Και τα δύο (parent + child) δείχνουν στα ίδια αρχεία. Γι' αυτό κλείνουμε αχρείαστα ends μετά από fork().
Κανόνας: ελάχιστος διαθέσιμος
Ο kernel ανοίγει πάντα το μικρότερο διαθέσιμο FD. Αν κλείσεις το 0 (stdin) και μετά ανοίξεις αρχείο, θα πάρει FD=0. Αυτό εκμεταλλευόμαστε με dup2().
📂

open() — Άνοιγμα Αρχείου

Η open() είναι η "πύλη" για την πρόσβαση σε αρχεία — το low-level αντίστοιχο της fopen(). Αντί για pointer επιστρέφει έναν File Descriptor (αριθμό) που χρησιμοποιούμε σε όλες τις επόμενες πράξεις. Πρέπει να ορίσουμε γιατί ανοίγουμε το αρχείο μέσω των flags: για ανάγνωση; για γράψιμο; να δημιουργηθεί αν δεν υπάρχει; να αδειάσει αν υπάρχει;

// Signatures int open(const char *path, int flags); int open(const char *path, int flags, mode_t mode); // επιστρέφει: fd ≥ 0 (επιτυχία) ή -1 (αποτυχία) // mode χρειάζεται ΜΟΝΟ με O_CREAT // παραδείγματα fd = open("data.txt", O_RDONLY); fd = open("out.txt", O_WRONLY|O_CREAT|O_TRUNC, 0644); fd = open("log.txt", O_WRONLY|O_CREAT|O_APPEND, 0644); fd = open("fifo", O_RDONLY|O_NONBLOCK);
O_RDONLY
Read Only — ανάγνωση μόνο
O_WRONLY
Write Only — εγγραφή μόνο
O_RDWR
Read+Write — και τα δύο
O_CREAT
CREATe — δημιουργία αν δεν υπάρχει
O_TRUNC
TRUNCate — άδειασμα αν υπάρχει
O_APPEND
APPEND — γράφει πάντα στο τέλος
O_EXCL
EXCLusive — αποτυχία αν ήδη υπάρχει
O_NONBLOCK
Non-blocking — δεν μπλοκάρει
🧠
Tip — flags από το όνομα
Κάθε flag περιγράφει τον εαυτό του: O_RDONLY = read-only, O_CREAT = create, O_TRUNC = truncate, O_APPEND = append. Τα συνδυάζεις με OR: O_WRONLY|O_CREAT|O_TRUNC = "ανοίξε για γράψιμο, δημιούργησε αν δεν υπάρχει, άδειασε αν υπάρχει".

read(), write(), lseek(), close()

Αφού ανοίξουμε ένα αρχείο, χρησιμοποιούμε αυτές τις συναρτήσεις για να μεταφέρουμε δεδομένα. Λειτουργούν με ωμά bytes — δεν υπάρχει "γραμμή" ή "τύπος": διαβάζεις/γράφεις N bytes. Ο δείκτης θέσης (file offset) προχωράει αυτόματα μετά από κάθε κλήση. Η lseek() σου επιτρέπει να τον τοποθετήσεις οπουδήποτε στο αρχείο (αρχή, τέλος, ή ενδιάμεση θέση).

// read() — διαβάζω bytes ssize_t read(int fd, void *buf, size_t count); // επιστρέφει: bytes διαβάστηκαν, 0=EOF, -1=error // write() — γράφω bytes ssize_t write(int fd, const void *buf, size_t n); // επιστρέφει: bytes γράφτηκαν, -1=error // lseek() — μετακινώ θέση ανάγνωσης/γραφής off_t lseek(int fd, off_t offset, int whence); // SEEK_SET=από αρχή, SEEK_CUR=τώρα+offset, SEEK_END=τέλος+offset lseek(fd, 0, SEEK_SET); // πήγαινε στην αρχή lseek(fd, 0, SEEK_END); // μάθε μέγεθος αρχείου // close() close(fd);
🧠
Tip — τιμές επιστροφής read()
read() επιστρέφει:
· > 0 = bytes που διαβάστηκαν
· = 0 = EOF (τέλος αρχείου / pipe κλείστηκε)
· = -1 = σφάλμα
Μνημονικό: "θετικό=δεδομένα, μηδέν=τέλος, αρνητικό=πρόβλημα"
// παράδειγμα: αντιγραφή αρχείου char buf[4096]; ssize_t n; while ((n = read(src_fd, buf, sizeof(buf))) > 0) { if (write(dst_fd, buf, n) != n) { perror("write"); exit(1); } } if (n == -1) { perror("read"); exit(1); }
⚠️ Partial reads/writes
read() μπορεί να επιστρέψει λιγότερα bytes απ' ότι ζήτησες (π.χ. network, pipe). Πάντα loop ή έλεγξε n == count.

dup() και dup2() — Η Τέχνη της Ανακατεύθυνσης

Οι συναρτήσεις dup επιτρέπουν την "κλωνοποίηση" ενός File Descriptor. Σκέψου τον FD table σαν έναν πίνακα από βέλη που δείχνουν σε αρχεία. Η dup2 αλλάζει το πού δείχνει ένα συγκεκριμένο βέλος.

// dup(oldfd): Κλωνοποιεί τον oldfd στον ΜΙΚΡΟΤΕΡΟ διαθέσιμο αριθμό. int dup(int oldfd); // dup2(oldfd, newfd): Η πιο "δυνατή" syscall. // Κάνει τον 'newfd' να δείχνει ΕΚΕΙ ΠΟΥ ΔΕΙΧΝΕΙ ο 'oldfd'. int dup2(int oldfd, int newfd);
💡 Αποκωδικοποίηση ονομάτων
  • 🔹 oldfd (Πηγή): Ο FD που "βλέπει" ήδη το αρχείο/pipe που μας ενδιαφέρει.
  • 🔹 newfd (Στόχος): Ο FD που θέλουμε να "καταλάβουμε" (π.χ. το 1 για stdout).
// ΠΑΡΑΔΕΙΓΜΑ: Ανακατεύθυνση του stdout σε αρχείο int fd = open("out.txt", O_WRONLY|O_CREAT|O_TRUNC, 0644); // 1. Κάνουμε το 1 (stdout) να δείχνει στο "out.txt" (εκεί που δείχνει ο fd) dup2(fd, 1); // 2. Κλείνουμε το αρχικό 'fd'. Δεν το χρειαζόμαστε πια, // αφού το '1' δείχνει πλέον στο ίδιο αρχείο. close(fd); printf("Γεια! Αυτό θα γραφτεί στο out.txt\n");
Βήμα-Βήμα Ανάλυση
  1. open(): Ο πυρήνας μας δίνει έναν FD (π.χ. 3) που δείχνει στο αρχείο out.txt.
  2. dup2(3, 1): Λέμε στον πυρήνα: "Πάρε το FD 1 και κάνε το να δείχνει στον ίδιο προορισμό με το FD 3". Αν το 1 ήταν ανοιχτό (που είναι), κλείνει αυτόματα.
  3. close(3): Κλείνουμε το "βοηθητικό" FD 3. Το FD 1 παραμένει να δείχνει στο αρχείο.
  4. printf(): Η printf γράφει πάντα στο FD 1. Επειδή το 1 δείχνει πλέον στο αρχείο, η έξοδος ανακατευθύνεται!

Tip για Pipes
Όταν θέλεις ένα παιδί να διαβάζει από pipe αντί για το πληκτρολόγιο: dup2(p[0], 0). Το 0 (stdin) γίνεται ο στόχος και το p[0] (pipe read) η πηγή.

stat() & opendir/readdir

Μερικές φορές χρειάζεσαι πληροφορίες για ένα αρχείο χωρίς να το ανοίξεις για ανάγνωση. Η stat() επιστρέφει τα metadata: μέγεθος, δικαιώματα, τύπο (αρχείο/κατάλογος/symlink), χρόνους δημιουργίας/τροποποίησης — ό,τι βλέπεις με ls -l. Αντίστοιχα, opendir() / readdir() σου επιτρέπουν να διατρέξεις τα περιεχόμενα ενός καταλόγου — αυτό κάνει εσωτερικά η ls.

// stat() — πληροφορίες για αρχείο #include <sys/stat.h> struct stat sb; if (stat("file.txt", &sb) == -1) { perror("stat"); exit(1); } printf("Size: %ld\n", sb.st_size); printf("Mode: %o\n", sb.st_mode); // octal printf("UID: %d\n", sb.st_uid); printf("Links: %ld\n", sb.st_nlink); // macros για τύπο αρχείου if (S_ISDIR(sb.st_mode)) { /* directory */ } if (S_ISREG(sb.st_mode)) { /* regular file */ }
🧠
Tip — st_ prefix
Όλα τα πεδία της struct stat αρχίζουν με st_ (abbreviation για "stat"): st_size, st_mode, st_uid, st_nlink. Εύκολο να θυμάσαι αν ξέρεις το prefix.
// opendir/readdir — διαβάζω κατάλογο #include <dirent.h> DIR *dir; struct dirent *entry; // opendir = open για directory dir = opendir("."); if (dir == NULL) { perror("opendir"); exit(1); } while ((entry = readdir(dir)) != NULL) { // παραλείπω . και .. if (strcmp(entry->d_name, ".") == 0 || strcmp(entry->d_name, "..") == 0) continue; printf("%s\n", entry->d_name); } closedir(dir);

Signals — Ασύγχρονα Events

Τα signals είναι ασύγχρονες ειδοποιήσεις — φαντάσου τα σαν "λογισμικές διακοπές" (software interrupts). Μπορούν να φτάσουν στη διεργασία οποιαδήποτε στιγμή και να διακόψουν την κανονική ροή της για να την αναγκάσουν να αντιδράσει σε ένα γεγονός (π.χ. Ctrl+C).

Τα signals είναι asynchronous notifications — μπορούν να φτάσουν οποιαδήποτε στιγμή, διακόπτοντας την εκτέλεση. Σαν interrupt από software.

Tip — SIGKILL και SIGSTOP
Αυτά τα δύο δεν μπορούν να πιαστούν, να αγνοηθούν ή να μπλοκαριστούν. Εκτελούνται πάντα από τον kernel. Υπάρχουν για να μπορεί ο admin να ελέγχει κάθε διεργασία.

sigaction() & sigprocmask() — Ο Έλεγχος των Σημάτων

Όταν ένα σήμα φτάνει στη διεργασία μας, η φυσιολογική ροή του προγράμματος διακόπτεται απότομα. Η sigaction και η sigprocmask είναι τα εργαλεία μας για να ελέγξουμε αυτή τη διακοπή.

🛠️ sigaction() — Ο "Επαγγελματίας" Handler

Η sigaction είναι η σύγχρονη και ασφαλής μέθοδος για να ορίσουμε έναν "χειριστή" (handler) για ένα σήμα. Αντίθετα με την παλιά signal(), μας επιτρέπει να ορίσουμε ακριβείς κανόνες.

struct sigaction sa; sa.sa_handler = my_handler; // Η συνάρτηση που θα τρέξει sigemptyset(&sa.sa_mask); // Καθάρισμα της μάσκας sa.sa_flags = SA_RESTART; // Μην σπας τα syscalls μου! sigaction(SIGINT, &sa, NULL); // Εγκατάσταση
Τι είναι τα πεδία της struct;
  • sa_handler: Η συνάρτηση "γιατρός" που θα αναλάβει το σήμα.
  • sa_mask: Λίστα με άλλα σήματα που θέλουμε να μπλοκάρουμε μόνο όσο τρέχει ο handler μας (για να μην διακοπεί ο ίδιος ο γιατρός!).
  • sa_flags: Ρυθμίσεις όπως το SA_RESTART (αν το σήμα διέκοψε μια read(), η read() θα συνεχίσει αυτόματα μετά).
🚫 sigprocmask() — Λειτουργία "Μην Ενοχλείτε"

Μερικές φορές έχουμε ένα κομμάτι κώδικα (Critical Section) που είναι πολύ ευαίσθητο και δεν θέλουμε να διακοπεί από κανένα σήμα (π.χ. όσο ανανεώνουμε μια κοινή λίστα).

sigset_t set, old; sigemptyset(&set); sigaddset(&set, SIGINT); // "Πρόσθεσε το SIGINT στη μαύρη λίστα" // 1. ΜΠΛΟΚΑΡΙΣΜΑ: Ξεκινά η "ησυχία" sigprocmask(SIG_BLOCK, &set, &old); /* --- Κρίσιμος Κώδικας εδώ --- */ // 2. ΕΠΑΝΑΦΟΡΑ: Επιστροφή στην κανονικότητα sigprocmask(SIG_SETMASK, &old, NULL);
Πώς λειτουργεί;
Ο πυρήνας κρατάει μια "μάσκα" (λίστα) για κάθε διεργασία. Αν ένα σήμα είναι στη μάσκα, ο πυρήνας το κρατάει "σε αναμονή" (pending) και θα το παραδώσει μόνο όταν το βγάλουμε από τη μάσκα.
⚠️ Ο Κανόνας της "Ασφάλειας" (Async-Signal-Safety)

Επειδή ένας handler μπορεί να τρέξει οποιαδήποτε στιγμή, μπορεί να διακόψει το πρόγραμμα ενώ αυτό χρησιμοποιεί μια "βαριά" συνάρτηση όπως η printf(). Αν ο handler καλέσει και αυτός printf(), το πρόγραμμα μπορεί να "κολλήσει" (deadlock).

❌ ΜΗΝ χρησιμοποιείς: printf(), malloc(), free(), exit()
✅ Χρησιμοποίησε: write(), _exit(), signal-safe syscalls
💡
Καλύτερη πρακτική: Στον handler απλά άλλαξε μια μεταβλητή τύπου volatile sig_atomic_t done = 1; και κάνε την πραγματική δουλειά στη main σου.
