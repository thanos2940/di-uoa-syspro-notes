Error Handling — Ο Χρυσός Κανόνας

Κάθε system call επιστρέφει -1 σε αποτυχία και θέτει τη global μεταβλητή errno με τον κωδικό σφάλματος. Ποτέ μην αγνοείς την επιστρεφόμενη τιμή.

// το ΣΩΣΤΟ pattern — πάντα έτσι int fd; if ((fd = open("file.txt", O_RDONLY)) == -1) { perror("open"); // ↑ εκτυπώνει: "open: No such file or directory" exit(1); } // ή με strerror για custom μήνυμα if (write(fd, buf, n) == -1) { fprintf(stderr, "write: %s\n", strerror(errno)); exit(1); }
⚠️ Headers
#include <errno.h> για errno · #include <string.h> για strerror() · #include <stdio.h> για perror()
🧠
Tip — perror vs strerror
perror("msg") = τυπώνει αυτόματα στο stderr: "msg: [error description]". Απλό και βολικό.
strerror(errno) = επιστρέφει string με description. Χρήσιμο αν θέλεις να το ενσωματώσεις σε δικό σου μήνυμα.
🚨 Κρίσιμο
Το errno ορίζεται μόνο αν η κλήση απέτυχε. Αν η κλήση πετύχει, η τιμή errno είναι undefined. Έλεγξε πάντα αν η κλήση απέτυχε πριν κοιτάξεις το errno.
📋

File Descriptors — Ο Πίνακας FD

Κάθε διεργασία έχει έναν πίνακα file descriptors. Κάθε FD είναι ένας μικρός ακέραιος που "δείχνει" σε ένα αρχείο/pipe/socket. Τα πρώτα 3 είναι πάντα ορισμένα:

PROCESS FD TABLE
0
→stdin — keyboard (terminal)
1
→stdout — screen (terminal)
2
→stderr — screen (terminal)
3
→1ο open() που κάνουμε
4
→2ο open(), κ.ο.κ.
🧠
Tip — γιατί 0, 1, 2
Είναι τα standard streams που κάθε UNIX διεργασία αποκτά αυτόματα όταν ξεκινά. Αντιστοιχούν στα stdin, stdout, stderr της C (αλλά εκεί είναι pointers, εδώ integers).
Κληρονομιά μετά από fork()
Το child process παίρνει αντίγραφο του FD table. Και τα δύο (parent + child) δείχνουν στα ίδια αρχεία. Γι' αυτό κλείνουμε αχρείαστα ends μετά από fork().
Κανόνας: ελάχιστος διαθέσιμος
Ο kernel ανοίγει πάντα το μικρότερο διαθέσιμο FD. Αν κλείσεις το 0 (stdin) και μετά ανοίξεις αρχείο, θα πάρει FD=0. Αυτό εκμεταλλευόμαστε με dup2().
📂

open() — Άνοιγμα Αρχείου

// Signatures int open(const char *path, int flags); int open(const char *path, int flags, mode_t mode); // επιστρέφει: fd ≥ 0 (επιτυχία) ή -1 (αποτυχία) // mode χρειάζεται ΜΟΝΟ με O_CREAT // παραδείγματα fd = open("data.txt", O_RDONLY); fd = open("out.txt", O_WRONLY|O_CREAT|O_TRUNC, 0644); fd = open("log.txt", O_WRONLY|O_CREAT|O_APPEND, 0644); fd = open("fifo", O_RDONLY|O_NONBLOCK);
O_RDONLY
Read Only — ανάγνωση μόνο
O_WRONLY
Write Only — εγγραφή μόνο
O_RDWR
Read+Write — και τα δύο
O_CREAT
CREATe — δημιουργία αν δεν υπάρχει
O_TRUNC
TRUNCate — άδειασμα αν υπάρχει
O_APPEND
APPEND — γράφει πάντα στο τέλος
O_EXCL
EXCLusive — αποτυχία αν ήδη υπάρχει
O_NONBLOCK
Non-blocking — δεν μπλοκάρει
🧠
Tip — flags από το όνομα
Κάθε flag περιγράφει τον εαυτό του: O_RDONLY = read-only, O_CREAT = create, O_TRUNC = truncate, O_APPEND = append. Τα συνδυάζεις με OR: O_WRONLY|O_CREAT|O_TRUNC = "ανοίξε για γράψιμο, δημιούργησε αν δεν υπάρχει, άδειασε αν υπάρχει".

read(), write(), lseek(), close()

// read() — διαβάζω bytes ssize_t read(int fd, void *buf, size_t count); // επιστρέφει: bytes διαβάστηκαν, 0=EOF, -1=error // write() — γράφω bytes ssize_t write(int fd, const void *buf, size_t n); // επιστρέφει: bytes γράφτηκαν, -1=error // lseek() — μετακινώ θέση ανάγνωσης/γραφής off_t lseek(int fd, off_t offset, int whence); // SEEK_SET=από αρχή, SEEK_CUR=τώρα+offset, SEEK_END=τέλος+offset lseek(fd, 0, SEEK_SET); // πήγαινε στην αρχή lseek(fd, 0, SEEK_END); // μάθε μέγεθος αρχείου // close() close(fd);
🧠
Tip — τιμές επιστροφής read()
read() επιστρέφει:
· > 0 = bytes που διαβάστηκαν
· = 0 = EOF (τέλος αρχείου / pipe κλείστηκε)
· = -1 = σφάλμα
Μνημονικό: "θετικό=δεδομένα, μηδέν=τέλος, αρνητικό=πρόβλημα"
// παράδειγμα: αντιγραφή αρχείου char buf[4096]; ssize_t n; while ((n = read(src_fd, buf, sizeof(buf))) > 0) { if (write(dst_fd, buf, n) != n) { perror("write"); exit(1); } } if (n == -1) { perror("read"); exit(1); }
⚠️ Partial reads/writes
read() μπορεί να επιστρέψει λιγότερα bytes απ' ότι ζήτησες (π.χ. network, pipe). Πάντα loop ή έλεγξε n == count.

dup() και dup2() — Ανακατεύθυνση FD

dup2() είναι από τις πιο εξεταζόμενες system calls. Κάνει τον ένα FD να δείχνει ακριβώς εκεί που δείχνει ο άλλος.

// dup(fd): επιστρέφει νέο fd που δείχνει στο ίδιο // (ελάχιστος διαθέσιμος αριθμός) int dup(int oldfd); // dup2(oldfd, newfd): κάνει newfd = oldfd // κλείνει αυτόματα το newfd αν ήταν ανοιχτό int dup2(int oldfd, int newfd); // παράδειγμα: ανακατεύθυνση stdout σε αρχείο int fd = open("out.txt", O_WRONLY|O_CREAT|O_TRUNC, 0644); if (fd == -1) { perror("open"); exit(1); } dup2(fd, 1); // fd=1 (stdout) → out.txt close(fd); // κλείνω το παλιό fd (δεν το χρειάζομαι) printf("Αυτό πάει στο out.txt!\n");

🧠
Tip — dup2 για pipes
Κλασική χρήση στα pipes: μετά fork(), child κάνει dup2(pipe[0], 0) για να διαβάζει από pipe μέσω stdin, και dup2(pipe[1], 1) για να γράφει στο pipe μέσω stdout. Μετά close τα αρχικά pipe ends.

stat() & opendir/readdir

// stat() — πληροφορίες για αρχείο #include <sys/stat.h> struct stat sb; if (stat("file.txt", &sb) == -1) { perror("stat"); exit(1); } printf("Size: %ld\n", sb.st_size); printf("Mode: %o\n", sb.st_mode); // octal printf("UID: %d\n", sb.st_uid); printf("Links: %ld\n", sb.st_nlink); // macros για τύπο αρχείου if (S_ISDIR(sb.st_mode)) { /* directory */ } if (S_ISREG(sb.st_mode)) { /* regular file */ }
🧠
Tip — st_ prefix
Όλα τα πεδία της struct stat αρχίζουν με st_ (abbreviation για "stat"): st_size, st_mode, st_uid, st_nlink. Εύκολο να θυμάσαι αν ξέρεις το prefix.
// opendir/readdir — διαβάζω κατάλογο #include <dirent.h> DIR *dir; struct dirent *entry; // opendir = open για directory dir = opendir("."); if (dir == NULL) { perror("opendir"); exit(1); } while ((entry = readdir(dir)) != NULL) { // παραλείπω . και .. if (strcmp(entry->d_name, ".") == 0 || strcmp(entry->d_name, "..") == 0) continue; printf("%s\n", entry->d_name); } closedir(dir);

Signals — Ασύγχρονα Events

Τα signals είναι asynchronous notifications — μπορούν να φτάσουν οποιαδήποτε στιγμή, διακόπτοντας την εκτέλεση. Σαν interrupt από software.

SIGKILL9terminateΆμεσος θάνατος — ο kernel εκτελεί✗ ΟΧΙ
SIGINT2terminateCtrl+C από terminal✓ ναι

// αποστολή signal kill(pid, SIGTERM); // σε συγκεκριμένο PID kill(0, SIGTERM); // σε ΟΛΟΚΛΗΡΗ την process group // alarm: στέλνει SIGALRM μετά N δευτερόλεπτα alarm(5); // SIGALRM σε 5 δευτ. alarm(0); // ακύρωση alarm // pause: κοιμάται ώσπου να έρθει signal pause();
// signal() — απλός handler void my_handler(int sig) { write(1, "caught!\n", 8); } signal(SIGINT, my_handler); // install handler signal(SIGINT, SIG_IGN); // αγνόηση signal(SIGINT, SIG_DFL); // επαναφορά default
🧠
Tip — SIGKILL και SIGSTOP
Αυτά τα δύο δεν μπορούν να πιαστούν, να αγνοηθούν ή να μπλοκαριστούν. Εκτελούνται πάντα από τον kernel. Υπάρχουν για να μπορεί ο admin να ελέγχει κάθε διεργασία.
⚙️

sigaction() και sigprocmask()

#include <signal.h> void handler(int sig) { // ΜΟΝΟ async-signal-safe εδώ! // write() = OK, printf() = ΟΧΙ safe write(1, "caught\n", 7); } int main() { struct sigaction sa; sa.sa_handler = handler; sigemptyset(&sa.sa_mask); // κανένα extra blocking sa.sa_flags = 0; // ή SA_RESTART if (sigaction(SIGINT, &sa, NULL) == -1) { perror("sigaction"); exit(1); } }
// sigprocmask — block signals σε critical section sigset_t newmask, oldmask; sigemptyset(&newmask); sigaddset(&newmask, SIGINT); // block SIGINT sigaddset(&newmask, SIGTERM); // block SIGTERM // critical section: block signals sigprocmask(SIG_BLOCK, &newmask, &oldmask); /* ... κώδικας που δεν πρέπει να διακοπεί ... */ // αποblock: επαναφορά sigprocmask(SIG_SETMASK, &oldmask, NULL);
🧠
Tip — signal() vs sigaction()
signal() = απλό, παλιό API. Η συμπεριφορά μπορεί να διαφέρει μεταξύ Unix. sigaction() = το σωστό για production: portable, predictable, περισσότερος έλεγχος. Στις εξετάσεις θα δεις και τα δύο — ξέρε αμφότερα.
⚠️ Signal handlers: async-safe only
Μέσα σε handler μπορείς μόνο async-signal-safe functions: write() ✓, printf() ✗. Χρησιμοποίησε volatile sig_atomic_t flag για communication μεταξύ handler και main.

🧠 Quick Check

Ποιο signal δεν μπορεί να πιαστεί ή αγνοηθεί;
SIGKILL (9) και SIGSTOP (19). Εκτελούνται πάντα από τον kernel ανεξάρτητα από handlers.
Τι κάνει η dup2(fd, 1);
Κάνει τον FD=1 (stdout) να δείχνει εκεί που δείχνει το fd. Αν fd είναι αρχείο, κάθε printf() στη συνέχεια γράφει στο αρχείο.
Τι επιστρέφει read() όταν φτάσει EOF;
0. Αρνητικό (-1) = error. Θετικό = bytes διαβάστηκαν.
Τι είναι το errno;
Global μεταβλητή που θέτει ο kernel με κωδικό σφάλματος αν η τελευταία system call απέτυχε. Αξία μόνο αμέσως μετά από αποτυχία — αλλάζει σε κάθε επόμενη κλήση.
Ποιοι είναι οι τρεις standard file descriptors και σε τι αντιστοιχούν;
0: stdin (πληκτρολόγιο), 1: stdout (οθόνη), 2: stderr (οθόνη για σφάλματα).
Τι κάνει ο συνδυασμός flags O_WRONLY | O_CREAT | O_TRUNC στην open();
Ανοίγει το αρχείο για γράψιμο μόνο, το δημιουργεί αν δεν υπάρχει, και αν υπάρχει ήδη, το "αδειάζει" (truncate) ώστε να ξεκινήσει από την αρχή.
Πότε χρειαζόμαστε το τρίτο όρισμα (mode) στην open();
Μόνο όταν χρησιμοποιούμε το flag O_CREAT, για να ορίσουμε τα permissions του νέου αρχείου (π.χ. 0644).
Τι κάνει η lseek(fd, 0, SEEK_END);
Μεταφέρει τον κέρσορα του αρχείου στο τέλος. Η τιμή που επιστρέφει αντιστοιχεί στο μέγεθος του αρχείου σε bytes.
Ποια η διαφορά μεταξύ stat() και lstat();
Αν το path είναι symbolic link, η stat() επιστρέφει πληροφορίες για το αρχείο στο οποίο δείχνει το link, ενώ η lstat() επιστρέφει πληροφορίες για το ίδιο το link.
Πώς ελέγχουμε αν ένα αρχείο είναι directory χρησιμοποιώντας τη struct stat;
Με το macro S_ISDIR(sb.st_mode).
Τι κάνει η alarm(5);
Προγραμματίζει τον πυρήνα να στείλει ένα σήμα SIGALRM στη διεργασία μετά από ακριβώς 5 δευτερόλεπτα.
Γιατί δεν πρέπει να καλούμε την printf() μέσα σε έναν signal handler;
Γιατί η printf() δεν είναι "async-signal-safe" (χρησιμοποιεί buffers και locks που μπορεί να οδηγήσουν σε deadlock αν διακοπεί η main). Χρησιμοποιούμε τη write() που είναι safe.
Τι είναι το sigset_t;
Είναι ένας τύπος δεδομένων (σύνολο) που χρησιμοποιείται για να αποθηκεύει πολλαπλά σήματα, π.χ. για να τα μπλοκάρουμε μαζικά με τη sigprocmask().
Τι κάνει η pause();
Αναστέλλει την εκτέλεση της διεργασίας μέχρι να φτάσει οποιοδήποτε σήμα που είτε την τερματίζει είτε καλεί έναν handler.
Ποιος είναι ο ρόλος του SA_RESTART στα flags της sigaction;
Αν ένα system call (π.χ. read) διακοπεί από ένα σήμα, ο πυρήνας θα το επανεκκινήσει αυτόματα μόλις τελειώσει ο handler, αντί να επιστρέψει σφάλμα EINTR.
Τι τύπου πρέπει να είναι μια global μεταβλητή που αλλάζει μέσα σε έναν handler;
Πρέπει να είναι volatile sig_atomic_t για να εγγυηθεί ο compiler ότι η πρόσβαση σε αυτήν είναι ατομική και δεν θα γίνει optimization που την κρύβει.
