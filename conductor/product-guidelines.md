# Product Guidelines: System Programming Web Notes & Quiz

## 1. Tone and Voice
- **Serious yet Accessible:** The content must maintain a level of academic seriousness appropriate for a university course, while remaining highly accessible.
- **Demystifying Concepts:** Complex system programming terms and concepts must be broken down using clear, relatable analogies.
- **Zero Prior Knowledge:** Explanations should be structured so that any student can grasp the topics (e.g., memory management, threading) without assuming extensive prior knowledge.

## 2. Visual Design & Branding
- **Interactive & Colorful:** The UI should leverage the existing dark theme but use distinct, vibrant colors to highlight different system states, code syntax, and interactive elements.
- **Visual Cues:** Colors should be used purposefully to denote success/failure in quizzes and to differentiate between various components (like parent vs. child processes).

## 3. User Experience (UX) Principles
- **Immediate Feedback:** Interactive quizzes must provide instantaneous feedback, clearly indicating correct or incorrect answers along with helpful explanations to reinforce learning.
- **Distraction-Free Reading:** The interface for the course notes should prioritize a clean, uncluttered reading experience, allowing students to focus entirely on the material without unnecessary visual noise.
