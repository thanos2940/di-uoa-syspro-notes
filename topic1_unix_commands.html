Αρχιτεκτονική Unix

Το Unix είναι δομημένο σε στρώματα. Η επικοινωνία user-space ↔ kernel γίνεται αποκλειστικά μέσω system calls.

📦 User Applications
ls, gcc, vim, browser…
↕
🐚 Shell (bash)
διερμηνεύει εντολές, scripting
↕
⚙️ System Calls
open() fork() read() write()…
↕
🔒 Kernel
διεργασίες · μνήμη · I/O · δίκτυο
↕
💾 Hardware
CPU · RAM · δίσκος
🧠
Πώς να το θυμάσαι
Σκέψου το σαν κρεμμύδι — στρώμα-στρώμα. Εσύ (user) είσαι απ' έξω. Ο kernel είναι στη μέση. Για να φτάσεις τον πυρήνα, περνάς μέσα από στρώματα. System calls = η πόρτα προς τον kernel.
💡 Φιλοσοφία Unix
"Do one thing and do it well." Κάθε εντολή κάνει μία δουλειά. Δύναμη από συνδυασμό με pipes.
Multi-user & Multi-tasking
Πολλοί χρήστες ταυτόχρονα, πολλές διεργασίες ταυτόχρονα, απομονωμένα μέσω UIDs και permissions.
man sections
Section 1: εντολές (ls, grep) · Section 2: system calls (open, fork) · Section 3: C library (printf, malloc). man 2 open → system call.

Σύστημα Αρχείων (Filesystem)

/ ← root, ρίζα ολόκληρου του δέντρου
├── bin/ binaries: ls, cp, mv, cat…
├── etc/ etcetera: config αρχεία
├── home/ home directories χρηστών
│ └── alice/
├── tmp/ temporary: χάνεται στο reboot
├── usr/ user programs (installed software)
├── var/ variable data: logs, spool…
└── dev/ device files (δίσκοι, τερματικά)
🧠
Absolute vs Relative path
Absolute = ξεκινά με / — "διεύθυνση από το σπίτι μου" (δουλεύει από παντού).
Relative = χωρίς / — "στρίψε αριστερά" (εξαρτάται από το πού είσαι τώρα).
.
Το τρέχον directory. Χρήσιμο ως ./prog για εκτέλεση.
🧠
Tip
. = "εδώ". Το shell δεν ψάχνει το CWD στο PATH για ασφάλεια, οπότε χρειάζεσαι ./ ρητά.
..
Ο γονικός κατάλογος. cd ../../foo = 2 επίπεδα πάνω → μπαίνει στο foo.
~
Home directory τρέχοντα χρήστη. cd χωρίς args = ίδιο με cd ~.
🧠
Tip
Το ~ μοιάζει οπτικά με στέγη/σπίτι. Εύκολο να το συνδέσεις με "home".

Εντολές Πλοήγησης & Αρχείων

pwdPrint Working Directory
Εκτυπώνει το απόλυτο path του τρέχοντος καταλόγου. Χρήσιμο όταν χάνεσαι στο filesystem.
$ pwd/home/alice/projects/k24
🧠
Tip — ακρωνύμιο
Print Working Directory. "Πού είμαι τώρα;" — ρωτάς το σύστημα.
lsLiSt
Λίστα αρχείων καταλόγου. Τα σημαντικότερα flags:
ls # απλή λίστα ls -l # long: permissions, owner, size, date ls -a # all: δείχνει κρυφά αρχεία (αρχίζουν με .) ls -la # συνδυασμός: long + all — το πιο χρήσιμο ls -lh # human-readable sizes (KB, MB αντί bytes) ls -R # recursive: μπαίνει σε υποφακέλους
🧠
Tip — τα flags
l = long (λεπτομέρειες), a = all (κρυφά), h = human (κατανοητά μεγέθη). Συνδυάζονται: ls -lah. Τα κρυφά αρχεία (dotfiles) αρχίζουν πάντα με ..
cdChange Directory
Αλλάζει τον τρέχοντα κατάλογο.
cd /home/alice # absolute path cd projects # relative path cd .. # ένα επίπεδο πάνω cd # πάντα στο home (~) cd - # εναλλαγή με τον προηγούμενο κατάλογο
🧠
Tip για το cd -
Σαν το κουμπί "back" στον browser. Χρήσιμο αν είσαι σε /deep/nested/path και θέλεις να επιστρέψεις γρήγορα.
mkdirMaKe DIRectory
Δημιουργεί κατάλογο.
mkdir mydirmkdir -p a/b/c/d # φτιάχνει ΟΛΑ τα ενδιάμεσα
🧠
Tip
-p = "parents". Χωρίς -p, αποτυγχάνει αν ο γονικός δεν υπάρχει.
cpCoPy
Αντιγράφει αρχεία/directories.
cp src.txt dst.txtcp -r dir1/ dir2/ # recursive για dirs
🧠
Tip
-r = recursive. Χρειάζεται για directories — χωρίς αυτό βγάζει error.
rmReMove
Διαγράφει αρχεία/directories. Μόνιμο — δεν πάει σε Recycle Bin.
rm file.txt rm -r mydir/ # recursive: διαγράφει directory με περιεχόμενο rm -f file.txt # force: δεν ρωτάει, δεν βγάζει error αν δεν υπάρχει rm -rf dir/ # recursive + force — ΠΡΟΣΟΧΗ
⚠️ Κανένα undo
rm -rf / ή rm -rf * διαγράφουν τα πάντα χωρίς επιβεβαίωση.
find
Βρίσκει αρχεία με κριτήρια. Syntax: find [path] [criteria] [action]
find . -name "*.c" # pattern matching στο όνομα find /tmp -type f # μόνο files (f=file, d=directory) find . -name "*.log" -delete # βρες ΚΑΙ διέγραψε find . -newer ref.txt # αρχεία νεότερα από ref.txt
🧠
Tip
Το πρώτο argument είναι πάντα το path αναζήτησης. Το . σημαίνει "από εδώ και κάτω". Μπορείς να βάλεις / για ολόκληρο το σύστημα.

Εντολές Επεξεργασίας Κειμένου

catconCATenate
Εκτυπώνει αρχεία στο stdout. Αρχικά φτιάχτηκε για συνένωση αρχείων.
cat file.txt # τυπώνει το αρχείο cat f1.txt f2.txt # τυπώνει f1 μετά f2 (concatenate!) cat f1.txt f2.txt > all.txt # συνενώνει σε νέο αρχείο
🧠
Tip — το όνομα λέει τα πάντα
catenate = συνένωση. Αν θυμάσαι ότι φτιάχτηκε για concatenation, θυμάσαι και γιατί δέχεται πολλά αρχεία ως arguments.
grepGlobal Regular Expression Print
Ψάχνει γραμμές που ταιριάζουν με pattern.
grep "error" log.txt # γραμμές με "error" grep -i "Error" log.txt # -i = insensitive (Error, error, ERROR) grep -n "main" prog.c # -n = number: δείχνει αριθμό γραμμής grep -r "TODO" . # -r = recursive: ψάχνει σε όλα τα αρχεία grep -v "ok" log.txt # -v = inVert: γραμμές που ΔΕΝ έχουν "ok"
🧠
Tip — τα flags
-i = case-insensitive · -n = number of line · -r = recursive · -v = invert (αντίστροφο). Κλασικό: ps aux | grep -v grep για να βρεις διεργασία χωρίς να περιλαμβάνεις το grep ίδιο.
head
Πρώτες N γραμμές (default 10).
head -20 file.txt # πρώτες 20
🧠
Tip
head = κεφάλι = αρχή. Αντίστοιχο: tail = ουρά = τέλος.
tail
Τελευταίες N γραμμές. Το -f παρακολουθεί real-time.
tail -f log.txt # live log monitoring
🧠
Tip
-f = follow. Παρακολουθεί το αρχείο καθώς γράφεται. Χρήσιμο για να βλέπεις logs live.
wcWord Count
Μετράει γραμμές, λέξεις, bytes.
wc -l f # lines · wc -w words · wc -c chars
🧠
Tip
lines / words / characters. Πιο χρήσιμο: command | wc -l για να μετρήσεις αποτελέσματα.
sort
Ταξινομεί γραμμές αλφαβητικά ή αριθμητικά.
sort -n f # αριθμητικά · -r = reverse
🧠
Tip — σημαντικό!
Χωρίς -n, το "10" έρχεται πριν το "2" (lexicographic)! Για αριθμούς πάντα sort -n.

Δικαιώματα (Permissions)

Κάθε αρχείο έχει 9 bits permissions: 3 για owner, 3 για group, 3 για others.

# ls -l output: -rwxr-xr-- 1 alice dev 2048 Apr 25 prog # 1ος χαρακτήρας: τύπος αρχείου # - = regular file # d = directory # l = symbolic link # r=4, w=2, x=1 (δυαδικό: 100, 010, 001) # rwx = 4+2+1 = 7 # r-x = 4+0+1 = 5 # r-- = 4+0+0 = 4
chmod 755 script.sh # octal notation chmod +x prog # symbolic: πρόσθεσε execute σε όλους chmod -w file.txt # αφαίρεσε write από όλους
🧠
Tip — το κόλπο με τα bits
Κάθε γράμμα = ένα bit:
r = 4 (100 binary), w = 2 (010 binary), x = 1 (001 binary).
Άθροισμα ανά ομάδα: rwx = 7, rw- = 6, r-x = 5, r-- = 4.
Μνημονικό: 4-2-1 σαν σειρά από μεγαλύτερο προς μικρότερο.
Num Binary Σύμβολο
7 111 rwx
6 110 rw-
5 101 r-x
4 100 r--
0 000 ---
✅ Συνηθισμένα
755 → scripts/dirs · 644 → κανονικά αρχεία · 600 → private (SSH keys) · 777 → ποτέ σε production

Redirections & Pipes

Κάθε διεργασία έχει 3 standard streams: 0 = stdin (keyboard), 1 = stdout (screen), 2 = stderr (screen, errors). Όλα μπορούν να ανακατευθυνθούν.

> file
stdout → αρχείο. Αντικαθιστά αρχείο.
ls > out.txt
>> file
stdout → αρχείο. Προσθέτει στο τέλος.
echo "log" >> log.txt
< file
stdin ← αρχείο αντί keyboard.
sort < names.txt
2> file
stderr → αρχείο (errors αποθηκεύονται).
gcc prog.c 2> err.txt
2>&1
stderr → εκεί που πάει το stdout.
make > all.log 2>&1
/dev/null
Bit bucket — πετάει την έξοδο τελείως.
cmd >/dev/null 2>&1
🧠
Tip — 2>&1 explained
Διάβαζε ως "κάνε τον fd 2 να δείχνει εκεί που δείχνει (&) ο fd 1". Η σειρά μετράει: > file 2>&1 (σωστό — stderr πάει στο file) vs 2>&1 > file (λάθος — stderr πάει στο αρχικό stdout).
|pipe operator
Συνδέει stdout μιας εντολής με stdin της επόμενης. Τρέχουν ταυτόχρονα (παράλληλα).
ps aux
|
grep python
|
awk '{print $2}'
|
sort -n
🧠
Tip — ο σωλήνας νερού
Σκέψου pipe ως σωλήνας νερού: νερό (δεδομένα) ρέει από αριστερά προς τα δεξιά. Κάθε εντολή παίρνει νερό, το επεξεργάζεται, και στέλνει στον επόμενο σωλήνα.

Διαχείριση Διεργασιών

psProcess Status
Εμφανίζει τρέχουσες διεργασίες.
ps # δικές μουps aux # όλες οι διεργασίες
🧠
Tip — aux
a = all users · u = user-oriented format · x = without terminal. Μαζί = πλήρης εικόνα.
kill
Στέλνει signal σε διεργασία (default: SIGTERM).
kill 1234 # SIGTERM: "παρακαλώ τερματίσου"kill -9 1234 # SIGKILL: force kill, δεν αγνοείται
🧠
Tip — SIGTERM vs SIGKILL
SIGTERM = ευγενικό αίτημα, η διεργασία μπορεί να αρνηθεί. SIGKILL (-9) = ο kernel σκοτώνει αμέσως, κανείς δεν μπορεί να το αποτρέψει. Πρώτα SIGTERM, μετά αν δεν απαντά, SIGKILL.
&background execution
Εκτελεί διεργασία στο background — δεν μπλοκάρει το terminal.
./prog & # εκτέλεση background[1] 12345 # job id [1] και PID 12345jobs # λίστα background jobsfg %1 # φέρε το job 1 στο foreground
🧠
Tip — Ctrl shortcuts
Ctrl+C → SIGINT (τερματισμός) · Ctrl+Z → suspend (παύση) → μετά bg για background ή fg για επιστροφή.

Μεταβλητές Περιβάλλοντος

# σημαντικές built-in μεταβλητές echo $PATH # directories για εκτελέσιμα echo $HOME # home directory echo $USER # τρέχων χρήστης echo $$ # PID τρέχοντος shell echo $? # exit code τελευταίας εντολής # ορισμός + export για κληρονομιά export MYVAR="hello"
🧠
Tip — exit codes
0 = επιτυχία, ≠0 = αποτυχία. Αντίστροφο από C! Λόγος: ένας τρόπος επιτυχίας, πολλοί τρόποι αποτυχίας (1=general error, 2=misuse, 127=not found…).
🧠
Tip — γιατί ./prog
Το . (CWD) δεν είναι στο PATH για ασφάλεια. Αλλιώς ένας κακόβουλος χρήστης θα έβαζε ψεύτικο ls στο CWD. Οπότε χρειάζεσαι ρητά ./.

🧠 Quick Check

Τι κάνει η εντολή ls -la 2>/dev/null;
Εκτελεί ls -la, τυπώνει αποτελέσματα (stdout) κανονικά, αλλά πετάει οποιοδήποτε error (stderr). Χρήσιμο αν δεν θέλεις "Permission denied" μηνύματα.
Διαφορά > και >>;
> αντικαθιστά (truncate + write), >> προσθέτει στο τέλος (append). Και τα δύο δημιουργούν το αρχείο αν δεν υπάρχει.
Τι τυπώνει echo $? αμέσως μετά από ls /nonexistent;
Μη-μηδενικό (συνήθως 2) — αποτυχία γιατί το directory δεν υπάρχει. Επιτυχία = 0.
Πώς βρίσκουμε όλες τις εμφανίσεις της λέξης "error" σε όλα τα αρχεία του τρέχοντος καταλόγου αναδρομικά;
Χρησιμοποιούμε grep -r "error" .. Το -r σημαίνει recursive και το . δηλώνει τον τρέχοντα κατάλογο.
Τι σημαίνει το permission 755 σε ένα αρχείο;
Owner: 7 (rwx), Group: 5 (r-x), Others: 5 (r-x). Ο ιδιοκτήτης κάνει τα πάντα, οι υπόλοιποι μόνο ανάγνωση και εκτέλεση.
Πώς εκτελούμε ένα πρόγραμμα myprog στον τρέχοντα κατάλογο και γιατί χρειάζεται το ./;
Με ./myprog. Το ./ είναι απαραίτητο γιατί ο τρέχων κατάλογος (.) δεν περιλαμβάνεται στο PATH για λόγους ασφαλείας.
Ποια η διαφορά μεταξύ kill (σκέτο) και kill -9;
Η kill στέλνει SIGTERM (αίτημα τερματισμού που μπορεί να αγνοηθεί), ενώ η kill -9 στέλνει SIGKILL (βίαιος τερματισμός από τον kernel που δεν αγνοείται).
Πώς ταξινομούμε ένα αρχείο με αριθμούς έτσι ώστε το "10" να έρχεται μετά το "2";
Χρησιμοποιούμε sort -n. Χωρίς το -n η ταξινόμηση είναι αλφαβητική και το "10" θα έμπαινε πριν το "2".
Σε ποιο section του manual βρίσκονται οι κλήσεις συστήματος (system calls) όπως η open;
Στο Section 2. Το Section 1 είναι για εντολές shell και το Section 3 για βιβλιοθήκες της C.
Τι κάνει η εντολή mkdir -p a/b/c;
Δημιουργεί τον κατάλογο c, καθώς και όλους τους ενδιάμεσους γονικούς καταλόγους (a, b) αν αυτοί δεν υπάρχουν ήδη.
Τι σημαίνει το σύμβολο ~ (tilde);
Αντιπροσωπεύει το Home Directory του τρέχοντος χρήστη (π.χ. /home/alice).
Πώς στέλνουμε και το stdout και το stderr στο ίδιο αρχείο all.log;
Με την ανακατεύθυνση command > all.log 2>&1. Το 2>&1 λέει στο stderr (2) να πάει εκεί που πάει το stdout (1).
Τι είναι το /dev/null;
Είναι μια ειδική "συσκευή" (bit bucket). Οτιδήποτε γράφεται εκεί απορρίπτεται αμέσως από το σύστημα. Χρησιμοποιείται για να "φιμώνουμε" την έξοδο εντολών.
Πώς βρίσκουμε όλα τα αρχεία .c και τα μετράμε;
Συνδυάζοντας εντολές με pipe: find . -name "*.c" | wc -l ή πιο απλά ls *.c | wc -l.
Πώς παρακολουθούμε τις τελευταίες γραμμές ενός log αρχείου σε πραγματικό χρόνο;
Χρησιμοποιούμε tail -f log.txt. Το flag -f (follow) περιμένει και τυπώνει νέες γραμμές καθώς προστίθενται.
